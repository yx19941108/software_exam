# 第17课：算法与代码专项

## 本课定位

第17课是下午案例题最后一个专项，目标不是背某一种算法的模板，而是训练“读懂题干算法说明 -> 读懂 C 代码骨架 -> 补关键语句 -> 判断算法策略与复杂度 -> 手工运行小样例”的完整能力。

本课只面向算法与代码案例题。设计模式、UML、数据库、DFD 等题型不纳入本课正文。算法题在本地完整案例题中通常出现在第4题，分值为15分，主要语言为 C，核心失分点集中在：看不懂数组含义、不会从递推式落到代码、复杂度不会按循环层数或递归规模判断、手工运行时下标错一位。

本课的学习顺序采取“先能读代码，再攻动态规划”的方式：先建立通用读题和填空方法，再重点突破出现频率最高、最容易丢分的动态规划题。

## 本课目标

学完本课后，需要达到以下目标：

1. 能识别下午算法题的四类常见问法：程序填空、算法策略、时间/空间复杂度、给定输入的运行结果或中间状态。
2. 能把题干中的变量说明、递归式、伪代码、步骤说明对应到 C 代码中的数组、循环、判断条件和返回值。
3. 能区分动态规划、回溯/深度优先搜索、分治、字符串匹配、排序、图遍历等算法考点。
4. 能用循环层数、递归缩减规模、主串/子串扫描次数判断常见复杂度。
5. 能手工跟踪小规模样例，写出数组状态、输出结果或最优值。
6. 对动态规划题形成固定动作：定义状态、找初始化、找转移、找填表顺序、找最终答案。

## 前置知识

### 1. C 代码基本读法

算法案例题通常不会考复杂语法，而是考你能不能读出代码的控制流程。需要先掌握：

- `for` 循环：看初始化、继续条件、每轮更新。复杂度常由循环次数决定。
- `while` 循环：看循环变量在哪里变化。如果循环变量没有变化，通常就是填空点。
- `if/else`：看条件成立时做什么，不成立时做什么。算法填空常考判断条件。
- 数组下标：题目可能从 `0` 开始，也可能从 `1` 开始。必须先看变量说明。
- 辅助单元：如 `R[0]` 常作为临时交换空间或哨兵，不一定是有效数据元素。
- 返回值：函数名和题干目标往往决定最终 `return` 应该返回哪个变量或数组单元。

### 2. 复杂度基本判断

复杂度不是靠背答案，而是看“规模为 n 时，关键操作执行多少次”。

- 单层循环从 `1` 到 `n`：通常是 `O(n)`。
- 双层循环都与 `n` 相关：通常是 `O(n^2)`。
- 三层循环都与 `n` 相关：通常是 `O(n^3)`。
- 每次把问题规模减半：通常是 `O(log n)`。
- 主串和子串各线性扫描一次：常见为 `O(lt + ls)`。
- 排序题要记常见结论：堆排序 `O(nlogn)`，希尔排序不稳定，复杂度通常优于直接插入但与增量序列有关。

### 3. 动态规划基本语言

动态规划不是“看到数组就是动态规划”。它的核心是：大问题的最优解可以由更小子问题的最优解组合得到，并且这些子问题会重复出现。

做动态规划题时必须问五个问题：

1. 状态是什么：例如 `c[i][j]` 表示前 `i` 个对象、容量 `j` 时的最优值。
2. 初始值是什么：例如第一行、第一列、长度为0、容量为0时的值。
3. 转移式是什么：例如从“不选第 i 个物品”和“选第 i 个物品”中取最大值。
4. 填表顺序是什么：代码是从小规模到大规模，还是递归自顶向下。
5. 答案在哪里：例如 `c[n][T]`、`d[len1][len2]`、`cost[0][n-1]`。

### 4. 回溯、DFS、分治的基本区别

- 回溯：在解空间中尝试一个选择，不行就撤回，再试下一个选择。关键词是“尝试、冲突、回退、所有方案、深度优先”。
- DFS：沿着一条路径一直访问到不能继续，再退回来访问其他路径。图题常用 `visited[]` 防止重复访问。
- 分治：把问题拆成若干更小的同类问题，分别求解后合并或比较。关键词是“划分、递归、规模减半”。

## 固定答题步骤

### 第一步：先读题干目标，不先钻代码

先用一句话写出算法要解决的问题，例如：

- 求两个字符串之间的某种最优值。
- 求排序后的中间状态。
- 求图的连通分量数。
- 求一个最优切分、装包、匹配或搜索方案。

如果目标没读懂，直接填代码通常会把条件写反。

### 第二步：圈出变量含义和下标范围

算法题中变量说明就是送分线索。重点圈：

- 数组代表什么。
- 有效下标从 `0` 还是 `1` 开始。
- `n`、`m`、`T`、`len1`、`len2` 分别表示什么规模。
- 临时变量是用来交换、保存最大值，还是保存当前位置。
- `visited[]`、`next[]`、`trace[]`、`cost[][]` 这类数组的语义。

特别注意：同样是二维数组，动态规划的 `d[i][j]` 可能表示“前 i 个和前 j 个的最优值”，图的 `c[i][j]` 可能表示“顶点 i 和顶点 j 是否相邻”，含义完全不同。

### 第三步：先做非代码填空题

如果题目同时问算法策略、复杂度、输出结果，建议先做这些题，因为它们能反过来帮助程序填空。

- 问“采用什么设计策略”：从题干特征判断。
- 问“时间复杂度”：先数循环层数或递归缩减方式。
- 问“输出/中间状态”：先手工模拟小样例。

这些答案不是附属题，而是理解代码的钥匙。

### 第四步：按“初始化、循环推进、判断条件、返回值”填代码

程序填空不要从第一个空硬推到底。按空的作用分类更稳：

1. 初始化空：通常出现在循环前或表格边界，如 `d[i][0]`、`count=0`。
2. 判断条件空：看 `if` 后的语句想做什么，判断条件就应该让它在正确场景下发生。
3. 循环推进空：`while` 中如果指针或下标没有变化，空里大概率是推进语句。
4. 递归/函数调用空：看注释和参数含义，补函数名与参数。
5. 返回值空：看题目最终要求的结果存在哪里。

### 第五步：手工运行必须写中间状态

手工运行题不要只在脑中跳结论。建议写三列：

| 轮次 | 当前操作 | 数组/变量状态 |
| --- | --- | --- |
| 初始 | 读入数据 | 写出原始数组或初始表 |
| 第1轮 | 执行关键交换/更新/转移 | 写出变化后的状态 |
| 第2轮 | 继续执行 | 写出变化后的状态 |

动态规划题则写小表；排序题写每一趟；DFS 题写访问序列和 `visited[]`；KMP 题写 `i`、`j`、`next[j]` 的变化。

### 第六步：堆排序题先判断下滤主形态

堆排序的核心思想只有一个：把一个节点和它的孩子比较，如果父节点不满足堆性质，就把它向下调整，直到以该节点为根的子树重新满足堆性质。考试不会要求背“交换式下滤”或“挖坑下滤法”这些名称，但会给出代码让你补空、判断建堆结果、判断第一次交换后结果。此时必须先判断代码形态，不能把一种写法的中间过程套到另一种写法上。

#### 形态一：交换式下滤

交换式下滤的特征是：每次发现父节点比较大的孩子小，就直接交换父节点和孩子，然后继续从孩子位置向下调整。

下面用 1 下标大顶堆写法示例。有效数据放在 `R[1]` 到 `R[n]`，`R[0]` 不参与堆。

```c
void Swap(int R[], int a, int b) {
    int temp;
    temp = R[a];
    R[a] = R[b];
    R[b] = temp;
}

void HeapifyBySwap(int R[], int v, int n) {
    int parent;
    int child;
    parent = v;
    child = parent * 2;

    while (child <= n) {
        if (child < n && R[child] < R[child + 1]) {
            child++;
        }
        if (R[parent] < R[child]) {
            Swap(R, parent, child);
            parent = child;
            child = parent * 2;
        } else {
            break;
        }
    }
}
```

交换式下滤每轮推演必须写清楚 4 件事：

1. `parent` 当前在哪个位置。
2. `child` 先指向左孩子，再根据 `child < n && R[child] < R[child + 1]` 改为较大的孩子。
3. 比较的是 `R[parent]` 和 `R[child]`。
4. 如果发生交换，数组立刻改变，下一轮从新的 `parent = child` 继续。

以 `R=(7,10,13,15,4,20,19,8)`、`n=8` 为例，完整建堆要从 `i = n / 2 = 4` 开始，依次调用 `HeapifyBySwap(R, i, n)`，直到 `i = 1`。下面是交换式下滤的完整建堆推演。

```text
初始：
R = (7,10,13,15,4,20,19,8)

第1次调用：i = 4，HeapifyBySwap(R, 4, 8)
parent = 4，child = 8。
R[4]=15，只有左孩子 R[8]=8，没有右孩子。
比较 R[parent]=R[4]=15 与 R[child]=R[8]=8。
15 >= 8，不交换，本次下滤结束。
R = (7,10,13,15,4,20,19,8)

第2次调用：i = 3，HeapifyBySwap(R, 3, 8)
parent = 3，child = 6。
R[6]=20，R[7]=19，较大的孩子是 R[6]，child = 6。
比较 R[parent]=R[3]=13 与 R[child]=R[6]=20。
13 < 20，交换 R[3] 与 R[6]。
R = (7,10,20,15,4,13,19,8)
parent = 6，child = 12。
child = 12 > n，本次下滤结束。

第3次调用：i = 2，HeapifyBySwap(R, 2, 8)
parent = 2，child = 4。
R[4]=15，R[5]=4，较大的孩子是 R[4]，child = 4。
比较 R[parent]=R[2]=10 与 R[child]=R[4]=15。
10 < 15，交换 R[2] 与 R[4]。
R = (7,15,20,10,4,13,19,8)
parent = 4，child = 8。
R[4]=10，只有左孩子 R[8]=8。
比较 R[parent]=R[4]=10 与 R[child]=R[8]=8。
10 >= 8，不交换，本次下滤结束。

第4次调用：i = 1，HeapifyBySwap(R, 1, 8)
parent = 1，child = 2。
R[2]=15，R[3]=20，较大的孩子是 R[3]，child = 3。
比较 R[parent]=R[1]=7 与 R[child]=R[3]=20。
7 < 20，交换 R[1] 与 R[3]。
R = (20,15,7,10,4,13,19,8)
parent = 3，child = 6。
R[6]=13，R[7]=19，较大的孩子是 R[7]，child = 7。
比较 R[parent]=R[3]=7 与 R[child]=R[7]=19。
7 < 19，交换 R[3] 与 R[7]。
R = (20,15,19,10,4,13,7,8)
parent = 7，child = 14。
child = 14 > n，本次下滤结束。

完整建堆结果：
R = (20,15,19,10,4,13,7,8)
```

如果考试代码是交换式，下滤过程中每次交换后数组都是真实变化；不能把原父节点存在脑中不动，也不能等到最后才落位。

#### 形态二：挖坑下滤法，也叫筛选法

挖坑下滤法的特征是：先把父节点保存到临时变量中，循环里不是交换父子节点，而是把较大的孩子一路向上提，最后把临时变量放到最终位置。

本地 `2022下半年案例题` 第4题就是这种写法。它使用 `R[0]` 保存原父节点。

```c
void HeapifyByHole(int R[], int v, int n) {
    int i;
    int j;

    i = v;
    j = 2 * i;
    R[0] = R[i];

    while (j <= n) {
        if (j < n && R[j] < R[j + 1]) {
            j++;
        }
        if (R[0] < R[j]) {
            R[i] = R[j];
            i = j;
            j = 2 * i;
        } else {
            j = n + 1;
        }
    }
    R[i] = R[0];
}
```

挖坑下滤每轮推演必须写清楚 5 件事：

1. `R[0]` 保存的是原父节点，后续比较一直拿 `R[0]` 和较大的孩子比。
2. `i` 是当前空位，也就是准备被填的位置。
3. `j` 先指向左孩子，再根据右孩子是否存在、是否更大，改为较大的孩子。
4. 若 `R[0] < R[j]`，说明孩子应上提，执行 `R[i] = R[j]`。
5. 循环结束后，必须执行 `R[i] = R[0]`，原父节点才真正落位。

以 `R=(7,15,20,10,4,13,19,8)`、`n=8`、从 `v=1` 调整为例，挖坑下滤推演如下：

```text
初始：
R = (7,15,20,10,4,13,19,8)
i = 1，j = 2
R[0] = R[1] = 7

第1轮：
R[2]=15，R[3]=20，较大的孩子是 R[3]，j = 3。
比较 R[0]=7 与 R[3]=20。
7 < 20，孩子 R[3] 上提到空位 R[1]。
R = (20,15,20,10,4,13,19,8)
i = 3，j = 6。

第2轮：
R[6]=13，R[7]=19，较大的孩子是 R[7]，j = 7。
比较 R[0]=7 与 R[7]=19。
7 < 19，孩子 R[7] 上提到空位 R[3]。
R = (20,15,19,10,4,13,19,8)
i = 7，j = 14。

第3轮：
j = 14 > n，循环结束。
执行 R[i] = R[0]，即 R[7] = 7。
R = (20,15,19,10,4,13,7,8)
```

中间状态里出现重复值不是错误。例如上面的 `20`、`19` 在临时数组中会短暂重复，因为旧位置还没有最终填回 `R[0]` 保存的值。只有最后 `R[i] = R[0]` 执行后，本轮下滤才结束。

#### 两种形态的考试识别对照

| 观察点 | 交换式下滤 | 挖坑下滤法 / 筛选法 |
| --- | --- | --- |
| 是否先暂存原父节点 | 通常不先整体暂存，只在交换时用 `temp` | 一开始就 `R[0] = R[i]` 或 `temp = R[i]` |
| 循环中数组如何变化 | 父节点和较大孩子直接交换 | 较大孩子上提到空位 |
| 每轮比较对象 | `R[parent]` 与 `R[child]` | `R[0]` 或 `temp` 与 `R[j]` |
| 原父节点何时落位 | 每次交换时已经换到孩子位置 | 循环结束后一次性落到最终位置 |
| 中间状态是否可能短暂重复 | 一般不会因写法产生重复 | 会，因为孩子先上提，旧位置暂未覆盖 |
| 最容易错的地方 | 交换后忘记更新 `parent` 和 `child` | 把 `R[i]` 当比较对象，忘记一直比较 `R[0]` |

#### 建堆与排序阶段不要混淆

堆排序通常分两阶段：

```c
for (i = n / 2; i >= 1; i--) {
    Heapify(R, i, n);
}

for (i = n; i > 1; i--) {
    swap_or_temp_exchange_root_and_tail;
    Heapify(R, 1, i - 1);
}
```

第一段是建堆。`n / 2` 是最后一个非叶子节点，因为 1 下标堆中节点 `i` 的左孩子是 `2i`，只有 `2i <= n` 才有孩子。叶子节点天然满足堆性质，不需要下滤。

第二段才是排序。每一轮把堆顶最大值放到当前堆的最后位置，再对剩余 `1` 到 `i - 1` 的元素重新调整成大顶堆。循环条件应是 `i > 1`，因为堆里只剩 1 个元素时已经有序。

#### 堆排序题每轮推演答题模板

正式作答时，建议每一轮都按这个模板写，避免把两种下滤法混用：

```text
当前阶段：建堆 / 排序后重调
当前调用：Heapify(R, v, n)
当前写法：交换式 / 挖坑式

初始：
写出 R、parent/i、child/j、temp/R[0]。

第1轮：
判断右孩子是否存在：child/j < n ?
选择较大的孩子：写出左右孩子数值。
比较对象：交换式写 R[parent] 与 R[child]；挖坑式写 R[0] 与 R[j]。
执行动作：交换 / 孩子上提 / 停止。
写出新数组。

第2轮：
更新 parent/i 与 child/j。
重复上述步骤。

结束：
交换式：child 越界或父节点已不小于孩子。
挖坑式：child 越界或 R[0] 已不小于孩子，最后写 R[i] = R[0]。
```

### 第七步：常考排序算法的完整代码、思想与变化过程

本节只覆盖软考软件设计师中最常见、最值得掌握的排序算法。排序题的核心不是背代码，而是看到题目代码后能判断：当前循环在确定哪个位置、哪个区间已经有序、每一趟结束后数组应该变成什么。

#### 排序算法性质总表

| 排序算法 | 最好时间 | 平均时间 | 最坏时间 | 额外空间 | 是否原地 | 是否稳定 | 软考常见考点 |

表中 `k` 表示关键字取值范围，`d` 表示关键字位数，`r` 表示基数。软考选择题常把“时间复杂度、空间复杂度、是否稳定、是否原地、适用场景”放在同一道题里考，必须和过程题一起掌握。

#### 1. 直接插入排序

思想：把数组分成“前面已排序区”和“后面未排序区”。每次取未排序区第一个元素，插入到前面已排序区的正确位置。已排序区中比它大的元素依次后移。

考试识别点：

- 代码中常有 `temp = A[i]`。
- 内层循环从 `i - 1` 往前走。
- 发现 `A[j] > temp` 时执行 `A[j + 1] = A[j]`。
- 最后执行 `A[j + 1] = temp`。
- 稳定；基本有序时效率高。

性质：

- 时间复杂度：最好 `O(n)`，平均和最坏 `O(n^2)`。
- 空间复杂度：`O(1)`。
- 原地排序：是。
- 稳定性：稳定，因为相等元素不会跨越移动。
- 适用场景：数据量较小或基本有序。

完整 C 代码：

```c
void InsertionSort(int A[], int n) {
    int i;
    int j;
    int temp;

    for (i = 1; i < n; i++) {
        temp = A[i];
        j = i - 1;
        while (j >= 0 && A[j] > temp) {
            A[j + 1] = A[j];
            j--;
        }
        A[j + 1] = temp;
    }
}
```

变化过程示例，升序排序 `A=(5,2,4,6,1)`：

```text
初始：        (5,2,4,6,1)
i=1，temp=2：5 后移，2 插入到 A[0]
第1趟后：     (2,5,4,6,1)
i=2，temp=4：5 后移，4 插入到 2 后面
第2趟后：     (2,4,5,6,1)
i=3，temp=6：前面元素都 <= 6，不移动
第3趟后：     (2,4,5,6,1)
i=4，temp=1：6、5、4、2 依次后移，1 插入到 A[0]
第4趟后：     (1,2,4,5,6)
```

#### 2. 希尔排序

思想：希尔排序是“分组的直接插入排序”。先用较大的步长 `gap` 把相隔 `gap` 的元素分成一组，在每组内做直接插入排序；然后逐步缩小 `gap`，最后 `gap = 1` 时做一次完整直接插入排序。

考试识别点：

- 代码中常有 `gap`、`delta`、`dk` 这类步长变量。
- 外层循环不断缩小步长，常见 `gap = gap / 2`。
- 内层不是 `j--`，而是 `j = j - gap`。
- 最后一趟 `gap = 1`。
- 不稳定。

性质：

- 时间复杂度：取决于增量序列；软考保守记忆为通常优于直接插入，最坏可按 `O(n^2)` 处理。
- 空间复杂度：`O(1)`。
- 原地排序：是。
- 稳定性：不稳定，因为相同关键字可能被分到不同 gap 组中，跨组移动后相对次序改变。
- 适用场景：中等规模、希望在插入排序基础上提升效率。

完整 C 代码：

```c
void ShellSort(int A[], int n) {
    int gap;
    int i;
    int j;
    int temp;

    for (gap = n / 2; gap > 0; gap = gap / 2) {
        for (i = gap; i < n; i++) {
            temp = A[i];
            j = i - gap;
            while (j >= 0 && A[j] > temp) {
                A[j + gap] = A[j];
                j = j - gap;
            }
            A[j + gap] = temp;
        }
    }
}
```

变化过程示例，升序排序 `A=(15,9,7,8,20,-1,4)`，`n=7`：

```text
初始：gap = n/2 = 3
A = (15,9,7,8,20,-1,4)

gap=3 时分组：
组1：A[0],A[3],A[6] = (15,8,4)，组内插入排序后为 (4,8,15)
组2：A[1],A[4] = (9,20)，排序后为 (9,20)
组3：A[2],A[5] = (7,-1)，排序后为 (-1,7)
还原到原数组位置：
A = (4,9,-1,8,20,7,15)

gap=1 时，对整个数组做直接插入排序：
最终：A = (-1,4,7,8,9,15,20)
```

本地 `2020下半年案例题` 考过希尔排序代码填空与第一趟排序结果。做这类题时，要先按步长分组，再还原到原数组位置。

#### 3. 冒泡排序

思想：相邻元素两两比较，逆序就交换。每一趟把当前未排序区的最大元素“冒”到末尾。

考试识别点：

- 内层比较的是 `A[j]` 和 `A[j + 1]`。
- 每趟结束，末尾多一个元素确定位置。
- 稳定。

性质：

- 时间复杂度：普通写法最好、平均、最坏均为 `O(n^2)`；带“本趟未交换则提前结束”标志的优化版最好为 `O(n)`。
- 空间复杂度：`O(1)`。
- 原地排序：是。
- 稳定性：稳定，因为只在 `A[j] > A[j + 1]` 时交换，相等不交换。
- 适用场景：教学和小规模数据；实际效率较低。

完整 C 代码：

```c
void BubbleSort(int A[], int n) {
    int i;
    int j;
    int temp;

    for (i = 0; i < n - 1; i++) {
        for (j = 0; j < n - 1 - i; j++) {
            if (A[j] > A[j + 1]) {
                temp = A[j];
                A[j] = A[j + 1];
                A[j + 1] = temp;
            }
        }
    }
}
```

变化过程示例，升序排序 `A=(49,38,65,97,76,13,27,49)`：

```text
初始：    (49,38,65,97,76,13,27,49)

第1趟：
49 与 38 交换 -> (38,49,65,97,76,13,27,49)
65 与 97 不换
97 与 76 交换 -> (38,49,65,76,97,13,27,49)

97 与 27 交换 -> (38,49,65,76,13,27,97,49)
97 与 49 交换 -> (38,49,65,76,13,27,49,97)
第1趟后最大值 97 归位。

第2趟：

49 与 65 不换
65 与 76 不换
76 与 13 交换 -> (38,49,65,13,76,27,49,97)
76 与 27 交换 -> (38,49,65,13,27,76,49,97)
76 与 49 交换 -> (38,49,65,13,27,49,76,97)
第2趟后： (38,49,65,13,27,49,76,97)
```

#### 4. 简单选择排序

思想：每一趟从未排序区中选出最小元素，放到未排序区的第一个位置。

考试识别点：

- 内层循环只找最小值下标 `min`，不频繁交换。
- 每趟最多交换一次。
- 不稳定。

性质：

- 时间复杂度：最好、平均、最坏均为 `O(n^2)`，因为每趟都要扫描未排序区。
- 空间复杂度：`O(1)`。
- 原地排序：是。
- 稳定性：不稳定，最小值与前面元素交换可能跨过相同关键字。
- 适用场景：交换代价高但比较代价低的小规模场景；软考更常考两趟过程和不稳定性。

完整 C 代码：

```c
void SelectionSort(int A[], int n) {
    int i;
    int j;
    int min;
    int temp;

    for (i = 0; i < n - 1; i++) {
        min = i;
        for (j = i + 1; j < n; j++) {
            if (A[j] < A[min]) {
                min = j;
            }
        }
        if (min != i) {
            temp = A[i];
            A[i] = A[min];
            A[min] = temp;
        }
    }
}
```

变化过程示例，升序排序 `A=(49,38,65,97,76,13,27,49)`：

```text
初始：    (49,38,65,97,76,13,27,49)

第1趟：
未排序区是全数组，最小值是 13。
交换 A[0]=49 与 13。
第1趟后：(13,38,65,97,76,49,27,49)

第2趟：
未排序区从 A[1] 开始，最小值是 27。
交换 A[1]=38 与 27。
第2趟后：(13,27,65,97,76,49,38,49)
```

选择排序与冒泡排序都可能考“两趟后序列”，区分方式是：冒泡每趟有很多相邻交换；选择排序每趟先找最小值，再交换一次。

#### 5. 快速排序

思想：选择一个基准 `pivot`，通过一趟划分把比基准小的元素放左边，比基准大的元素放右边，然后递归处理左右子区间。快速排序采用分治法。

考试识别点：

- 常考“一趟划分后数组是什么”。
- 基准可能取第一个、最后一个或中间元素，必须按题干给定。
- 平均 `O(nlogn)`，最坏 `O(n^2)`。
- 不稳定。

性质：

- 时间复杂度：最好和平均 `O(nlogn)`；最坏 `O(n^2)`，例如每次划分极不平衡。
- 空间复杂度：平均递归栈 `O(logn)`；最坏递归栈 `O(n)`。
- 原地排序：通常称为原地划分，但严格说递归栈不是 `O(1)`。
- 稳定性：不稳定，划分交换会改变相同关键字相对顺序。
- 适用场景：平均性能好；软考常考一趟划分、分治策略和最坏情况。

完整 C 代码，写法一：最后一个元素作基准：

```c
int PartitionLastPivot(int A[], int left, int right) {
    int pivot;
    int i;
    int j;
    int temp;

    pivot = A[right];
    i = left - 1;
    for (j = left; j < right; j++) {
        if (A[j] <= pivot) {
            i++;
            temp = A[i];
            A[i] = A[j];
            A[j] = temp;
        }
    }
    temp = A[i + 1];
    A[i + 1] = A[right];
    A[right] = temp;
    return i + 1;
}

void QuickSortLastPivot(int A[], int left, int right) {
    int mid;
    if (left < right) {
        mid = PartitionLastPivot(A, left, right);
        QuickSortLastPivot(A, left, mid - 1);
        QuickSortLastPivot(A, mid + 1, right);
    }
}
```

变化过程示例，`A=(2,8,7,1,3,5,6,4)`，以最后一个元素 `4` 为基准：

```text
初始：pivot=4，i=-1
A = (2,8,7,1,3,5,6,4)

j=0，A[0]=2 <= 4，i=0，A[0] 与 A[0] 自交换
A = (2,8,7,1,3,5,6,4)
j=1，8 > 4，不动
j=2，7 > 4，不动
j=3，1 <= 4，i=1，交换 A[1] 与 A[3]
A = (2,1,7,8,3,5,6,4)
j=4，3 <= 4，i=2，交换 A[2] 与 A[4]
A = (2,1,3,8,7,5,6,4)
j=5，5 > 4，不动
j=6，6 > 4，不动
循环结束，交换 A[i+1]=A[3] 与 pivot A[7]
一趟划分后：
A = (2,1,3,4,7,5,6,8)
```

完整 C 代码，写法二：第一个元素作基准，左右双指针挖坑：

```c
int PartitionFirstPivot(int A[], int left, int right) {
    int pivot;
    int i;
    int j;

    pivot = A[left];
    i = left;
    j = right;
    while (i < j) {
        while (i < j && A[j] >= pivot) {
            j--;
        }
        A[i] = A[j];
        while (i < j && A[i] <= pivot) {
            i++;
        }
        A[j] = A[i];
    }
    A[i] = pivot;
    return i;
}
```

这个写法和堆排序挖坑式类似：先保存基准，左右两边交替把元素填到空位，最后基准落位。做题时要按题目代码确定比较方向。

#### 6. 归并排序

思想：先把数组不断二分，直到每个子数组只有一个元素；再把两个有序子数组合并成一个更大的有序数组。归并排序也采用分治法。

考试识别点：

- 稳定。
- 时间复杂度稳定为 `O(nlogn)`。
- 需要额外数组，空间复杂度通常为 `O(n)`。
- 可能考两个有序序列合并过程或比较次数。

性质：

- 时间复杂度：最好、平均、最坏均为 `O(nlogn)`。
- 空间复杂度：数组归并通常为 `O(n)`。
- 原地排序：通常不是原地排序。
- 稳定性：稳定，合并时相等元素优先取左侧即可保持原相对顺序。
- 适用场景：要求稳定且时间复杂度保证 `O(nlogn)`；外部排序也常用归并思想。

完整 C 代码：

```c
void Merge(int A[], int temp[], int left, int mid, int right) {
    int i;
    int j;
    int k;

    i = left;
    j = mid + 1;
    k = left;
    while (i <= mid && j <= right) {
        if (A[i] <= A[j]) {
            temp[k] = A[i];
            i++;
        } else {
            temp[k] = A[j];
            j++;
        }
        k++;
    }
    while (i <= mid) {
        temp[k] = A[i];
        i++;
        k++;
    }
    while (j <= right) {
        temp[k] = A[j];
        j++;
        k++;
    }
    for (i = left; i <= right; i++) {
        A[i] = temp[i];
    }
}

void MergeSort(int A[], int temp[], int left, int right) {
    int mid;
    if (left < right) {
        mid = (left + right) / 2;
        MergeSort(A, temp, left, mid);
        MergeSort(A, temp, mid + 1, right);
        Merge(A, temp, left, mid, right);
    }
}
```

变化过程示例，合并两个有序序列 `A=(1,4,7)`、`B=(2,3,6,8)`：

```text
比较 1 和 2，取 1 -> R=(1)
比较 4 和 2，取 2 -> R=(1,2)
比较 4 和 3，取 3 -> R=(1,2,3)
比较 4 和 6，取 4 -> R=(1,2,3,4)
比较 7 和 6，取 6 -> R=(1,2,3,4,6)
比较 7 和 8，取 7 -> R=(1,2,3,4,6,7)
A 已取完，复制 B 剩余 8
最终 R=(1,2,3,4,6,7,8)
```

#### 7. 堆排序完整代码

思想：先把数组建成大顶堆，再反复把堆顶最大值放到当前堆的最后位置，并对剩余堆重新下滤调整。堆排序不稳定，时间复杂度 `O(nlogn)`，额外空间 `O(1)`。

性质：

- 时间复杂度：最好、平均、最坏均为 `O(nlogn)`；建堆是 `O(n)`，之后 `n-1` 次下滤合计 `O(nlogn)`。
- 空间复杂度：`O(1)`。
- 原地排序：是。
- 稳定性：不稳定，堆顶与末尾交换会改变相同关键字相对顺序。
- 适用场景：要求最坏时间也为 `O(nlogn)` 且希望额外空间为 `O(1)`；TopK 类问题也常用堆。

交换式完整 C 代码，1 下标版本：

```c
void SwapInt(int R[], int a, int b) {
    int temp;
    temp = R[a];
    R[a] = R[b];
    R[b] = temp;
}

void HeapifySwap(int R[], int v, int n) {
    int parent;
    int child;

    parent = v;
    child = 2 * parent;
    while (child <= n) {
        if (child < n && R[child] < R[child + 1]) {
            child++;
        }
        if (R[parent] < R[child]) {
            SwapInt(R, parent, child);
            parent = child;
            child = 2 * parent;
        } else {
            break;
        }
    }
}

void HeapSortSwap(int R[], int n) {
    int i;
    for (i = n / 2; i >= 1; i--) {
        HeapifySwap(R, i, n);
    }
    for (i = n; i > 1; i--) {
        SwapInt(R, 1, i);
        HeapifySwap(R, 1, i - 1);
    }
}
```

挖坑式完整 C 代码，1 下标版本：

```c
void HeapifyHole(int R[], int v, int n) {
    int i;
    int j;

    i = v;
    j = 2 * i;
    R[0] = R[i];
    while (j <= n) {
        if (j < n && R[j] < R[j + 1]) {
            j++;
        }
        if (R[0] < R[j]) {
            R[i] = R[j];
            i = j;
            j = 2 * i;
        } else {
            break;
        }
    }
    R[i] = R[0];
}

void HeapSortHole(int R[], int n) {
    int i;
    for (i = n / 2; i >= 1; i--) {
        HeapifyHole(R, i, n);
    }
    for (i = n; i > 1; i--) {
        R[0] = R[i];
        R[i] = R[1];
        R[1] = R[0];
        HeapifyHole(R, 1, i - 1);
    }
}
```

完整变化过程示例，`R=(7,10,13,15,4,20,19,8)`，先建大顶堆：

```text
建堆后：
R = (20,15,19,10,4,13,7,8)

第1轮排序：
交换堆顶 20 与末尾 8：
R = (8,15,19,10,4,13,7,20)
调整前 7 个元素：
R = (19,15,13,10,4,8,7,20)

第2轮排序：
交换堆顶 19 与 R[7]=7：
R = (7,15,13,10,4,8,19,20)
调整前 6 个元素：
R = (15,10,13,7,4,8,19,20)

第3轮排序：
交换堆顶 15 与 R[6]=8：
R = (8,10,13,7,4,15,19,20)
调整前 5 个元素：
R = (13,10,8,7,4,15,19,20)

继续重复，最终升序：
R = (4,7,8,10,13,15,19,20)
```

#### 8. 计数排序与基数排序只掌握场景

计数排序思想：当关键字是范围较小的整数时，统计每个值出现次数，再按计数输出。适合“关键字取值范围小”的题。

基数排序思想：按个位、十位、百位等关键字位进行多轮分配和收集。适合定长数字或字符串。软考中通常考适用场景，不要求下午写完整 C 代码。

性质：

- 计数排序时间复杂度：`O(n+k)`，空间复杂度 `O(n+k)` 或 `O(k)`，取决于是否需要输出辅助数组。
- 计数排序是否原地：通常不是。
- 计数排序稳定性：可稳定实现，关键看输出时是否按稳定规则回填。
- 基数排序时间复杂度：`O(d(n+r))`，空间复杂度 `O(n+r)`。
- 基数排序是否原地：通常不是。
- 基数排序稳定性：每一位分配收集若使用稳定排序，则整体稳定。

排序题优先级：

```text
下午代码级：希尔排序、堆排序。
上午过程级：冒泡、简单选择、快速排序划分、归并合并、堆建堆。
性质判断级：稳定性、时间复杂度、空间复杂度、适用场景。
```

## 判分点模板

### 1. 程序填空题

常见 8 分左右。每空通常 2 分。判分不看你会不会背算法名，而看语句是否能让程序逻辑闭合。

答题模板：

- 空(1)：说明这是初始化/判断/推进/返回中的哪一种。
- 空(2)：写出表达式或语句。
- 如有等价写法，优先写最贴近源代码变量名的写法。

常见扣分：

- 把 `i` 和 `j` 写反。
- 忘记数组从 `0` 或 `1` 开始。
- 条件写成赋值，如把 `==` 写成 `=`。
- 只写变量名，不写完整语句。
- 返回局部中间值，而不是最终答案位置。

### 2. 算法策略题

常见 2-4 分。不能只写“动态规划法”就结束，讲评时要能说出依据。

答题模板：

- 该问题采用：某算法策略。
- 判断依据：题干是否出现最优子结构、重复子问题、回溯尝试、规模折半、线性匹配、图遍历等特征。

识别提示：

- 有二维表、递归式、最优子结构：优先考虑动态规划。
- 有“尝试，不行退回”：优先考虑回溯。
- 有 `visited[]` 和邻接表/邻接矩阵：优先考虑图遍历。
- 有主串、子串、`next[]`：优先考虑 KMP。
- 有规模减半或分组称重：优先考虑分治。

### 3. 复杂度题

常见 2-4 分。必须用主规模表达，不能写具体次数。

答题模板：

- 单层主循环：`O(n)`。
- 双层主循环：`O(n^2)` 或 `O(mn)`。
- 三层主循环：`O(n^3)`。
- 每次规模减半：`O(logn)`。
- KMP 主串和子串线性处理：`O(lt + ls)`。
- 堆排序：`O(nlogn)`。

### 4. 手工运行题

常见 3-5 分。答案必须符合题目要求的格式：数组、字符串、最优值、访问序列、括号化计算顺序等。

答题模板：

1. 写出初始状态。
2. 按代码顺序执行关键步骤。
3. 每次改变数组、表格、计数器时记录。
4. 最后按题目格式写结果。

## 常见失分点

1. 看到“最优”就直接写动态规划。实际上分治、贪心、回溯也可能求某种结果，必须结合递归式、表格和中间结果存储判断。
2. 程序填空只看当前行，不看上下文。算法题的空常常需要看前后 5-10 行。
3. 忽略注释。下午算法题的注释经常直接说明函数、变量、数组的意义。
4. 动态规划表初始化漏掉第一行或第一列。
5. 复杂度把输入样例的规模当成通用规模。
6. 手工运行排序时把“数组位置”和“堆结构位置”混在一起。
7. DFS 题忘记更新指针或忘记标记 `visited`，导致死循环或重复访问。
8. KMP 题把 `next[]` 当成普通下标数组，不理解它是在失配时决定模式串回退位置。
9. 写答案时没有使用题目变量名，导致等价思路难以判分。
10. 堆排序题把交换式下滤和挖坑下滤混用。交换式每轮比较当前父节点并直接交换；挖坑式每轮比较临时保存的原父节点，并让孩子上提，最后才落位。

## 过关标准

第17课过关不以“听懂讲解”为准，而以能独立完成以下动作作为标准：

1. 看到一段 C 算法题，能在 3 分钟内说出题目目标、核心数组含义、下标起点。
2. 对程序填空，至少能先判断每个空属于初始化、判断、推进、调用、返回中的哪一类。
3. 动态规划题能写出：状态、初始化、转移、填表顺序、答案位置。
4. 排序或图遍历题能写出至少两轮中间状态。
5. 复杂度题能给出 `O()` 表达式，并能用循环层数或递归规模解释。
6. 第一轮训练目标：先拿到 9 分以上；低于 9 分时不进入下一类算法，先补代码阅读和手工运行。
7. 动态规划专项目标：连续两道动态规划案例题达到 12 分以上，才认为本课核心高频考点初步过关。

## 后续训练安排

第一轮选用本地完整度高、题干和解析齐全的排序类案例题作为入门训练，重点训练数组、循环、交换、复杂度和手工运行。后续建议顺序为：

1. 排序/堆结构：建立代码阅读与数组状态跟踪能力。
2. 图遍历/DFS：训练指针推进、`visited[]`、递归调用。
3. 动态规划入门：编辑距离或最长公共子串。
4. 动态规划进阶：0-1背包、矩阵链乘。
5. 回溯与分治：八皇后、假币问题。
