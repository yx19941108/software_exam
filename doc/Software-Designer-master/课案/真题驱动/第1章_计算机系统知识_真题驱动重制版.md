## 一、存储体系、总线与校验码（权值：5/20，约25.0%）

### 2.1 这个知识点到底在讲什么

这一块研究的是计算机最底层的信息怎么被存、怎么被搬、怎么被校验。

题目最常考三条线：

1. 不同层次存储器谁快谁慢
2. 总线带宽怎么算
3. CRC / 海明码怎么判断

### 2.2 核心概念详细讲解

#### 2.2.1 存储层次

按访问速度由快到慢，大致可以这样理解：

- 寄存器
- Cache
- 主存
- 辅存

其中：

- `寄存器` 离 CPU 最近，访问速度最快
- `Cache` 是主存和 CPU 之间的高速缓冲
- `主存` 是程序运行时主要工作的内部存储
- `辅存` 是硬盘、SSD 这类外部大容量存储

所以一旦题目问“CPU 访问最快的是谁”，优先想到：

- 寄存器

#### 2.2.2 Cache 映像方式

Cache 与主存之间常见映像方式：

- 直接映像
- 组相联映像
- 全相联映像

直观理解：

- 直接映像：规则最死，冲突也最容易发生
- 组相联：在规则和灵活性之间折中
- 全相联：最灵活，块冲突通常最少

所以若题目问：

- “发生块冲突次数最少的是哪种映像”

一般优先选：

- 全相联映像

#### 2.2.3 总线带宽

总线带宽本质上就是：

- 单位时间能传多少数据

常用思路：

1. 看总线宽度：每次传多少 bit
2. 看时钟频率：每秒有多少周期
3. 看多少周期传一次数据
4. 最后统一单位，换成 `MB/s`

#### 2.2.4 CRC

CRC 是循环冗余校验。

它的特点是：

- 擅长检错
- 不负责纠错
- 通过生成多项式计算余数，得到校验位

看到 CRC，优先想到：

- 检错
- 多项式
- 余数

#### 2.2.5 海明码

海明码是典型的纠错码。

它能做的事是：

- 发现 1 位错误
- 纠正 1 位错误

海明码最常考的是校验位个数。

公式是：

- 若信息位数为 `m`，校验位数为 `r`
- 则满足 `2^r >= m + r + 1`

例如：

- `32` 位信息位，至少要多少校验位？
- 从 `r=5` 开始试：
  - `2^5 = 32`
  - 但 `m + r + 1 = 38`
  - 不够
- `r=6`：
  - `2^6 = 64`
  - `64 >= 39`
  - 成立

所以 `32` 位数据至少需要 `6` 位校验位。

### 2.3 易错点

1. 把 `Cache` 当成主存。
2. 把 `CRC` 和 `海明码` 混为一谈。
3. 海明码题不会代公式试最小 `r`。
4. 总线带宽题不统一 bit / Byte / MB 单位。

### 2.4 真题示例 1

题目：

以下关于 Cache 与主存间地址映射的叙述中，正确的是（D）。

- A. 操作系统负责管理 Cache 与主存之间的地址映射
- B. 程序员需要通过编程来处理 Cache 与主存之间的地址映射
- C. 应用软件对 Cache 与主存之间的地址映射进行调度
- D. 由硬件自动完成 Cache 与主存之间的地址映射

正确答案：`D`

详细解析：

- Cache 与主存之间的地址映射属于硬件层面机制。
- 程序员和应用软件不会逐条去“安排 Cache 映射”。
- 所以 A、B、C 都错，D 对。

### 2.5 真题示例 2

题目：

海明码是一种纠错码……对于 `32` 位的数据，至少需要增加（D/B）个校验位才能构成海明码。

问题1：

- A. 3
- B. 4
- C. 5
- D. 6

正确答案：`D`

详细解析：

- 用海明码公式：`2^r >= m + r + 1`
- 这里 `m = 32`
- 试 `r = 5`：
  - `2^5 = 32`
  - `32 + 5 + 1 = 38`
  - 不满足
- 试 `r = 6`：
  - `2^6 = 64`
  - `32 + 6 + 1 = 39`
  - 满足

所以最少是 `6` 位。

## 二、中断、DMA 与输入输出控制（权值：4/20，约20.0%）

### 3.1 这个知识点到底在讲什么

这一块考的是 CPU 和外设怎么配合。

最常见的三种 I/O 控制方式：

1. 程序查询
2. 中断
3. DMA

### 3.2 核心概念详细讲解

`程序查询`：

- CPU 主动反复问设备是否准备好了
- CPU 开销大

`中断`：

- 设备准备好后通知 CPU
- CPU 暂停当前程序，处理中断服务程序
- 然后返回原程序继续执行

`DMA`：

- 数据块直接在主存和外设之间传输
- CPU 只在开始和结束时参与

中断题常见两个小点：

- `保存现场`
- `中断向量`

保存现场的目的是：

- 中断处理完以后，能正确返回被打断的程序继续执行

中断向量提供的是：

- 中断服务程序入口地址

### 3.3 易错点

1. 误以为 DMA 完全不需要 CPU。
2. 误以为保存现场是为了保护中断程序自己。
3. 误以为中断向量给的是设备地址。

### 3.4 真题示例 1

题目：

以下关于中断方式与 DMA 方式的叙述中，正确的是（A）。

- A. 中断方式与DMA方式都可实现外设与CPU之间的并行工作
- B. 程序中断方式和DMA方式在数据传输过程中都不需要CPU的干预
- C. 采用DMA方式传输数据的速度比程序中断方式的速度慢
- D. 程序中断方式和DMA方式都不需要CPU保护现场

正确答案：`A`

详细解析：

- 中断方式下，CPU 不必一直等设备。
- DMA 方式下，CPU 不亲自搬运每个数据块。
- 所以两者都能实现一定程度的并行。
- B 错：中断方式仍需要 CPU 处理中断；DMA 也需要 CPU 启动和收尾。
- C 错：DMA 通常更适合大块传输。
- D 错：中断处理中必须保存现场。

### 3.5 真题示例 2

题目：

计算机运行过程中，进行中断处理时需保存现场，其目的是（C）。

- A. 防止丢失中断处理程序的数据
- B. 防止对其他程序的数据造成破坏
- C. 能正确返回被中断的程序继续执行
- D. 能为中断处理程序提供所需的数据

正确答案：`C`

详细解析：

- 现场保存的是“被打断程序当时的运行状态”。
- 只有保存了这些信息，中断完成后 CPU 才能回到正确断点继续执行。
- 所以 C 对。

## 三、数值表示、补码与浮点数（权值：4/20，约20.0%）

### 4.1 这个知识点到底在讲什么

这一块考的是机器如何表示数字。

高频点有：

- 补码
- 定点数
- 浮点数
- 对阶

### 4.2 核心概念详细讲解

`补码` 最重要的特点：

- 零表示唯一
- 减法可以转成加法
- 符号位也参与运算

对于定点纯小数，常见结论是：

- 补码能精确表示 `-1`

`浮点数` 的本质表示为：

- `尾数 × 基数^阶码`

浮点加法前最重要的步骤是：

- 对阶

对阶规则：

- 小阶向大阶对齐
- 尾数右移

### 4.3 真题示例

题目：

浮点数的表示分为阶和尾数两部分。两个浮点数相加时，需要先对阶，即（D）（`n` 为阶差的绝对值）。

- A. 尾数不变，同时将较大的阶码加 `n`
- B. 尾数不变，同时将较小的阶码减 `n`
- C. 将较小的阶码对应的尾数左移 `n`
- D. 将较小的阶码对应的尾数右移 `n`

正确答案：`D`

详细解析：

- 两个浮点数相加前，必须先把阶码对齐。
- 对齐原则是：`小阶向大阶看齐`。
- 小阶本身不能凭空变大，只能通过把对应尾数右移来实现数值等价调整。
- 所以 D 正确。
- C 错在方向反了，左移会把数放大。

这类题的核心句子一定要记住：

- `浮点加法先对阶，小阶尾数右移。`

## 四、流水线、RISC/CISC 与可靠性（权值：4/20，约20.0%）

### 5.1 这个知识点到底在讲什么

这部分考的是体系结构层面的经典概念：

- 流水线性能
- RISC / CISC 特征
- 串并联可靠性

### 5.2 核心概念详细讲解

`流水线`

- 吞吐率通常取决于最慢流水段
- 节拍由最长阶段决定

`RISC`

- 指令少而简单
- 指令长度常固定
- 寄存器多
- 寻址方式少
- 便于流水线

`CISC`

- 指令复杂
- 寻址方式多
- 一条指令功能可能更多

`可靠性`

- 串联：可靠性相乘
- 并联：用 `1 - (1 - R)^n` 这类思路算“至少一个成功”

### 5.3 真题示例

题目：

以下关于 RISC 和 CISC 的叙述中，不正确的是（B）。

- A. RISC 的大多指令在一个时钟周期内完成
- B. RISC 普遍采用微程序控制器，CISC 则普遍采用硬布线控制器
- C. RISC 的指令种类和寻址方式相对于 CISC 更少
- D. RISC 和 CISC 都采用流水线技术

正确答案：`B`

详细解析：

- A 对：RISC 指令通常更精简，更容易在较少周期完成。
- B 错：说反了。通常是 `RISC 更常配合硬布线控制`，`CISC 更常配合微程序控制`。
- C 对：RISC 的一大特点就是指令种类和寻址方式更少。
- D 对：两者都可以采用流水线，只是 RISC 更适合。

这道题的真正考点不是只认识缩写，而是记住两组关键词：

- `RISC = 指令简单、寻址少、寄存器多、便于流水线`
- `CISC = 指令复杂、寻址多、功能更强`

## 五、密码学、数字签名与网络安全机制（权值：3/20，约15.0%）

### 6.1 这个知识点到底在讲什么

第1章里的安全基础更偏“计算机系统层面”的通识：

- 对称加密 / 摘要 / 签名
- 数字证书
- HTTPS 的握手与传输职责
- 访问控制

### 6.2 核心概念详细讲解

`数字证书`：

- 用来绑定“身份 + 公钥”

`数字签名`：

- 保证来源真实性
- 保证内容完整性
- 保证不可否认性

常见算法判断：

- `DES`：对称加密
- `SHA / MD5`：摘要算法
- `RSA`：非对称算法，可用于数字签名

`HTTPS`

- 握手阶段：证书校验、密钥协商
- 传输阶段：一般用会话密钥做对称加密

### 6.3 真题示例

题目：

可用于数字签名的算法是（A）。

- A. RSA
- B. IDEA
- C. RC4
- D. MD5

正确答案：`A`

详细解析：

- RSA 是非对称加密算法，可以做签名和验签。
- IDEA、RC4 属于对称加密。
- MD5 是摘要算法，不是签名算法。
