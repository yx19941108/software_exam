## 一、函数依赖、候选键与规范化（权值：8/22，约36.4%）

### 2.1 这个知识点到底在讲什么

这是数据库客观题里最稳定的高频板块，核心在于判断：

- 哪些属性组合能唯一标识元组
- 哪些依赖是冗余的
- 当前关系能达到几范式

### 2.2 核心概念详细讲解

`函数依赖`：

- 如果属性集 `X` 的值一旦确定，属性集 `Y` 的值也就确定，就称 `X → Y`

`候选键`：

- 能唯一标识元组
- 并且再去掉任何一个属性就不再唯一

`主属性`：

- 出现在候选键中的属性

`规范化`：

- 目的是消除冗余、异常与不合理依赖

常见考点：

- 全码是什么意思
- 是否存在传递依赖
- Armstrong 公理与冗余依赖

### 2.3 易错点

1. 把“能唯一标识”误写成“主键一定只有一个属性”。
2. 不会从函数依赖推出候选键。
3. 看到 3NF 就只记定义，不会结合依赖关系判断。

### 2.4 真题示例

题目：

给定关系模式 `R（U，F）`，其中：`U={A1，A2，A3，A4}`，`F={A1→A2，A1A2→A3，A1→A4，A2→A4}`，那么关系 `R` 的主键应为（A/C）。函数依赖集 `F` 中的（ ）是冗余的。

问题1：

- A. `A1`
- B. `A1A2`
- C. `A1A3`
- D. `A2A3`

正确答案：`A`

详细解析：

- 因为 `A1→A2`，又有 `A1A2→A3`，所以 `A1` 能进一步推出 `A3`。
- 同时 `A1→A4` 已经给出。
- 所以 `A1` 就能唯一决定全体属性，是候选键。
- 因此问题1选 `A`。
- 这类题的标准做法就是：从某属性出发，一步步闭包展开，看最终能否覆盖所有属性。

## 二、数据模型、关系代数与 SQL（权值：6/22，约27.3%）

### 3.1 这个知识点到底在讲什么

这一块关注的是数据库“怎么表示、怎么查询”。

常见考法：

- 关系代数表达式
- SQL 子句作用
- 视图定义

### 3.2 核心概念详细讲解

关系代数常见操作：

- 选择 `σ`：筛行
- 投影 `π`：取列
- 连接 `⋈`：把有关联的表连起来

SQL 中常见考点：

- `WHERE`：分组前过滤
- `GROUP BY`：分组
- `HAVING`：分组后过滤

### 3.3 真题示例

题目：

在 SQL 查询中，用于过滤分组后数据的子句是（B）。

- A. `GROUP BY`
- B. `HAVING`
- C. `ORDER BY`
- D. `WHERE`

正确答案：`B`

详细解析：

- `WHERE` 是分组前过滤原始行。
- `HAVING` 是对分组后的结果再做筛选。
- `GROUP BY` 负责分组，不负责过滤。
- `ORDER BY` 负责排序。

所以分组后过滤只会是 `HAVING`。

## 三、事务、并发控制与恢复（权值：4/22，约18.2%）

### 4.1 这个知识点到底在讲什么

事务考的是数据库如何保证：

- 多个操作要么全做、要么全不做
- 多人同时访问时数据不乱

### 4.2 核心概念详细讲解

事务常见关键字：

- `COMMIT`：提交
- `ROLLBACK`：回滚

一旦提交：

- 更改生效
- 一般就不能再靠回滚撤销

并发控制则是为了解决：

- 丢失更新
- 不可重复读
- 脏读

### 4.3 真题示例

题目：

在 SQL 中，结束事务通常可以使用 `COMMIT` 和 `ROLLBACK` 语句。若某事务 `T` 执行了（A）。

- A. `ROLLBACK` 语句，则可将 `T` 对数据库的更新撤销
- B. `ROLLBACK` 语句，则可将 `T` 对数据库的更新写入数据库
- C. `COMMIT` 语句，则 `T` 对数据库影响可用 `ROLLBACK` 语句来撤销
- D. `ROLLBACK` 语句，则表示 `T` 已正确地执行完毕

正确答案：`A`

详细解析：

- `ROLLBACK` 的作用是撤销事务，把数据库恢复到事务开始前的状态，所以 A 正确。
- B 错：把更新写入数据库对应的是提交，不是回滚。
- C 错：一旦 `COMMIT`，事务修改正式生效，通常不能再用 `ROLLBACK` 撤销。
- D 错：回滚不是“执行完毕”，而是“放弃这次事务修改”。

所以事务题最该记住的两句话是：

- `COMMIT = 提交生效`
- `ROLLBACK = 撤销未提交修改`

## 四、数据库设计与分布式数据库（权值：4/22，约18.2%）

### 5.1 这个知识点到底在讲什么

这部分关注的是：

- E-R 图如何转关系模式
- 数据库概念结构设计步骤
- 分布式数据库有哪些透明性和特征

### 5.2 真题示例

题目：

在数据库逻辑设计阶段，若实体中存在多值属性，那么将 `E-R` 图转换为关系模式时，（C），得到的关系模式属于 `4NF`。

- A. 将所有多值属性组成一个关系模式
- B. 使多值属性不在关系模式中出现
- C. 将实体的码分别和每个多值属性独立构成一个关系模式
- D. 将多值属性和其它属性一起构成该实体对应的关系模式

正确答案：`C`

详细解析：

- 多值属性如果直接塞进原关系，会导致冗余和异常。
- 正确做法是把“实体主键 + 多值属性”单独拆成一个关系模式。
- 这正是数据库逻辑设计中的典型分解思路。
