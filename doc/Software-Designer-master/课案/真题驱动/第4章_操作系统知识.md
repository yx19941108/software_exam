## 2. 第四课正文

## 一、设备管理（权值：21/76，约27.6%）

### 2.1 这个知识点到底在讲什么

设备管理研究的是：操作系统怎样管理键盘、鼠标、打印机、磁盘、扫描仪这类外部设备，让 CPU 和外设高效配合。

你可以把它理解成两层问题：

1. 数据怎么在 `CPU / 主存 / 外设` 之间流动。
2. 多个设备请求同时到来时，系统按什么规则先处理谁。

所以设备管理题的高频点基本都围绕这两件事展开：

- I/O 控制方式
- 中断
- DMA
- I/O 软件分层
- 磁盘调度
- 旋转延迟与传输时间

### 2.2 核心概念详细讲解

#### 2.2.1 I/O 控制方式

最常见的三种 I/O 控制方式是：

1. 程序查询方式
2. 中断方式
3. DMA 方式

程序查询方式的意思是：

- CPU 反复检查设备是否准备好。
- 设备没准备好，CPU 也要一直等。
- 所以 CPU 参与最多，效率最低。

中断方式的意思是：

- CPU 先去做别的事。
- 外设准备好以后，发出中断请求。
- CPU 暂停当前程序，转去执行中断服务程序。
- 处理完再返回原程序。

键盘题最常在这里出陷阱：

- 键盘输入速度慢
- 每次输入量很小
- 什么时候按键并不确定

所以管理键盘最适合用中断方式，而不是 DMA。

- 中断方式：有键按下时再通知 CPU，CPU 平时不用一直干等
- DMA 方式：更适合磁盘、网卡这类高速成块传输
- 程序查询方式：CPU 要不断轮询“键盘有没有准备好”，浪费时间

DMA 方式的意思是：

- DMA 控制器直接在主存和外设之间搬运一整块数据。
- CPU 只在开始时发启动命令，结束时做收尾处理。
- 所以 DMA 特别适合块传输。

这三种方式怎么区分，考试里最常见的判断表是：

| 比较项 | 程序查询 | 中断方式 | DMA方式 |
| --- | --- | --- | --- |
| CPU 是否一直参与 | 是 | 否 | 否 |
| 适合单个数据还是成块数据 | 少量数据 | 中等 | 成块数据 |
| CPU 与外设能否并行 | 很弱 | 可以 | 更强 |
| 谁负责实际搬运数据 | CPU | CPU/中断服务程序 | DMA 控制器 |

所以题里只要看到：

- “成块传输”
- “CPU 只首尾参与”
- “主存与外设直接交换”

大概率就是 DMA。

#### 2.2.2 什么是中断，为什么要保存现场

中断的本质是：

- CPU 正在执行当前程序时
- 突然收到一个更需要马上处理的事件
- 于是暂停当前程序
- 转去处理中断服务程序
- 处理完再回来继续执行原程序

所谓“保存现场”，就是把当前程序运行到一半时的重要信息先保存下来，比如：

- 程序计数器
- 通用寄存器内容
- 标志位

为什么必须保存？

- 因为中断处理结束后，CPU 还要准确回到被打断的位置继续执行。
- 如果不保存现场，就不知道原程序执行到哪了，也可能把原程序数据破坏掉。

#### 2.2.3 什么是中断向量

中断向量最容易被问成一句话：

- 中断向量提供什么？

标准答案是：

- 中断服务程序的入口地址。

你可以这样记：

- 设备发中断请求，只是说“我有事”。
- CPU 真正要去处理时，必须知道“去哪里执行处理程序”。
- 中断向量就是这个“地址指针”。

它不是：

- 设备地址
- 主程序断点地址
- 待传送数据的起始地址

#### 2.2.4 I/O 软件分层

I/O 软件题常考“分几层、每层干什么”。

可以粗记成四层：

1. 用户层 I/O 软件
2. 与设备无关的系统软件
3. 设备驱动程序
4. 中断处理程序

理解时不要死背名字，先抓职责：

- 用户层：用户程序发起 I/O 请求。
- 与设备无关层：做统一接口、缓冲、设备命名等通用管理。
- 驱动层：把通用请求翻译成具体设备能懂的命令。
- 中断处理层：处理中断响应和现场切换。

这里有一个非常稳定的判断句：

- I/O 软件对上层“隐藏实现细节”
- 对用户提供的是“逻辑上统一的接口”

所以考试里如果看到“开放了 I/O 实现细节”这类表述，通常就是错的。

你应该把 I/O 软件理解成：

- 向上：让程序员不用关心具体设备寄存器和时序
- 向下：把不同设备的具体差异封装起来

#### 2.2.5 磁盘调度到底在调什么

磁盘调度分两步：

1. 先移臂
   - 让磁头移动到目标柱面
2. 再旋转
   - 等目标扇区转到磁头下面

所以顺序一定是：

- 先移臂调度
- 再旋转调度

这一点如果不搞清，磁盘题特别容易做错。

#### 2.2.6 常见磁盘调度算法

`FCFS`：

- 先来先服务
- 按请求到达顺序处理
- 实现简单
- 但磁头可能来回大幅摆动

`SSTF`：

- 最短寻道时间优先
- 每次找离当前磁头最近的柱面
- 平均性能通常比 FCFS 好
- 但可能反复变方向，也可能导致远处请求长期得不到服务

`SCAN`：

- 电梯调度
- 磁头朝一个方向移动，沿路处理请求，走到尽头再回头
- 方向相对稳定

`CSCAN`：

- 单向扫描
- 只朝一个方向服务，到头后快速返回起点再重新扫描
- 更公平，但回卷本身不服务请求

所以考试里问“哪个可能随时改变方向”，通常是：

- FCFS
- SSTF

而不是：

- SCAN
- CSCAN

### 2.3 解题步骤

设备管理题推荐按下面顺序拆：

1. 先判断题目问的是 `中断 / DMA / I/O 控制 / 磁盘调度 / I/O层次` 哪一类。
2. 如果问控制方式，就看 CPU 是不是全程参与。
3. 如果问中断，就看是否需要保存现场、是否返回原程序。
4. 如果问 DMA，就看是不是“主存与外设直接成块传输”。
5. 如果问磁盘调度，就先分清移臂调度和旋转调度。

### 2.4 易错点

1. 误以为 DMA 完全不需要 CPU。错，DMA 只是不需要 CPU 逐字逐字搬数据，开始和结束还是要 CPU 配合。
2. 误以为中断向量是设备地址。错，中断向量给的是中断服务程序入口地址。
3. 误以为磁盘调度先旋转后移臂。错，先移臂，再旋转。
4. 误以为电梯调度也会像 SSTF 一样随时折返。错，电梯调度方向相对固定。

### 2.5 真题示例 1

题目：

以下关于中断方式与 DMA 方式的叙述中，正确的是（A）。

- A. 中断方式与DMA方式都可实现外设与CPU之间的并行工作
- B. 程序中断方式和DMA方式在数据传输过程中都不需要CPU的干预
- C. 采用DMA方式传输数据的速度比程序中断方式的速度慢
- D. 程序中断方式和DMA方式都不需要CPU保护现场

正确答案：`A`

详细解析：

- A 为什么对：
  - 中断方式下，CPU 发出 I/O 请求后可以去做别的事，外设准备好时再发中断。
  - DMA 方式下，CPU 不亲自搬每个数据，DMA 控制器负责块传输。
  - 所以两者都能让 CPU 与外设并行工作。
- B 为什么错：
  - 中断方式仍然需要 CPU 进中断服务程序处理。
  - DMA 方式开始和结束阶段也需要 CPU 配合。
- C 为什么错：
  - DMA 通常比程序中断方式更适合大块数据传输，不会更慢。
- D 为什么错：
  - 中断发生时必须保存现场，否则无法返回原程序继续执行。

这道题真正考的不是一句定义，而是你能不能分清：

- 中断方式：CPU 不一直等，但要处理中断。
- DMA 方式：CPU 不直接搬数据，但不是完全不参与。

### 2.6 真题示例 2

题目：

磁盘调度分为移臂调度和旋转调度两类，在移臂调度的算法中，（D）算法可能会随时改变移动臂的运行方向。

- A. 单向扫描和先来先服务
- B. 电梯调度和先来先服务
- C. 电梯调度和最短寻道时间优先
- D. 先来先服务和最短寻道时间优先

正确答案：`D`

详细解析：

- FCFS 为什么会变向：
  - 它只按请求到达顺序处理。
  - 若后到请求刚好在磁头反方向，磁头就会折返。
- SSTF 为什么会变向：
  - 它总选最近柱面。
  - 最近柱面可能在左，也可能在右，所以方向不固定。
- SCAN 和单向扫描为什么一般不变向：
  - 它们的核心思想就是沿固定方向扫过去。
  - 只有扫到边界时才统一掉头或回卷，不是“随时变向”。

这道题真正考的是你是否理解“算法行为”，不是背简称。

### 2.7 本节结论

- 设备管理是第四课最重的板块。
- 一定要把 `中断 / DMA / 中断向量 / 磁盘调度` 吃透。

## 二、文件管理（权值：16/76，约21.1%）

### 3.1 这个知识点到底在讲什么

文件管理研究的是：

- 文件怎么组织
- 路径怎么表示
- 磁盘块怎么分配
- 文件内容怎么定位

题库里最稳定的四类考法是：

1. 绝对路径与相对路径
2. 位示图
3. 索引节点
4. 多级索引

### 3.2 核心概念详细讲解

#### 3.2.1 绝对路径与相对路径

绝对路径：

- 从根目录开始写
- 不依赖当前工作目录

相对路径：

- 从当前工作目录开始写
- 不从根开始

路径题的标准做法是：

1. 先找“当前工作目录”。
2. 再找目标文件所在位置。
3. 从根写一遍，就是绝对路径。
4. 从当前目录写一遍，就是相对路径。

很多人错不是不会，而是第一步没找准起点。

另外，文件管理里有一个很容易和“路径”并列出现的高频点：目录文件本身为什么重要。

- 目录文件不是普通用户数据，它保存的是“文件名、层级关系、文件位置”这类元信息。
- 如果用户数据文件写坏，通常只是那一个文件受影响。
- 如果目录文件写坏，系统可能连“文件在哪里”都找不到。

所以考试里如果问“哪类文件写回磁盘时崩溃，对系统影响更大”，目录文件往往比普通用户数据更危险。

#### 3.2.2 位示图到底是什么

位示图是文件系统管理“整个磁盘空间是否空闲”的办法。

你可以把它理解成一张“全盘占用状态表”：

- 一个物理块对应一位
- 这一位只回答一个问题：这个物理块现在是空闲还是占用
- 所以位示图解决的是“哪些块还能分配”的问题

它和索引节点不是一回事：

- 位示图站在“整个文件系统”的角度，管理全盘空闲块
- 索引节点站在“某一个文件”的角度，管理这个文件自己的数据块地址

题目里如果出现“字长为 32 位、64 位、128 位”，意思不是让你去想 CPU 指令，而是：

- 位示图会按“字”成组存放
- 一个字能装多少位，就能一次描述多少个物理块

例如字长为 `32` 位时：

- 第 `0` 号字描述物理块 `0~31`
- 第 `1` 号字描述物理块 `32~63`
- 第 `2` 号字描述物理块 `64~95`

所以位示图题通常有两类：

1. 已知磁盘总容量和块大小，求整张位示图要多大
2. 已知某个物理块号，反推它落在位示图的第几个字、第几位

第二类题最容易错在“编号从 `0` 开始”。

例如：

- 物理块号 `2053`
- 字长 `32` 位

计算方法固定是：

- 字编号 = `2053 // 32 = 64`
- 位编号 = `2053 % 32 = 5`

这表示：

- 2053 号物理块落在“第 `64` 号字”
- 它对应这个字里的第 `5` 号位

为什么很多人会错选 `65`？

- 因为脑子里把它想成了“第 65 个字”
- 但题目问的是“编号为多少的字”
- 既然编号从 `0` 开始，第 65 个字的编号就是 `64`

所以位示图计算题总是分两条路线：

1. 如果问整张位示图大小：
   - 先求物理块总数：`磁盘容量 / 块大小`
   - 再求总位数：`物理块总数`
   - 再求多少个字：`总位数 / 每字位数`
2. 如果问某个块落在哪个字：
   - 先用 `块号 // 每字位数` 求字编号
   - 再用 `块号 % 每字位数` 求位编号

常见错误：

- 容量单位没统一
- 把“字长”当成字节数
- 把“第几个字”误写成“编号是多少”
- 混淆“位示图管理全盘空闲块”和“索引节点管理单个文件地址”

#### 3.2.3 索引节点是什么

索引节点可以理解为“文件自己的地址目录”。

它记录的不是文件正文，而是：

- 这个文件被分成了哪些逻辑块
- 每个逻辑块对应磁盘上的哪个物理块

这里一定要先分清两个编号：

- 逻辑块号：站在“文件内部”看的编号，比如这个文件的第 `0` 块、第 `1` 块、第 `2` 块
- 物理块号：站在“整个磁盘”看的编号，比如磁盘的第 `18` 块、第 `207` 块、第 `999` 块

所以索引节点真正做的事情是：

- 把“文件内部的逻辑块号”
- 映射成“磁盘上的物理块号”

一句话记忆：

- 位示图管“还有哪些块空着”
- 索引节点管“这个文件已经用了哪些块”

#### 3.2.4 直接索引、一级间接索引、二级间接索引

直接索引：

- 索引项直接指向数据块
- 访问最快

一级间接索引：

- 索引项先指向一个“索引块”
- 这个索引块里再存很多个“数据块地址”

二级间接索引：

- 索引项先指向一个“二级索引块”
- 二级索引块里存的是“一级索引块地址”
- 一级索引块里再存“数据块地址”
- 最后才能找到真正数据块

所以索引层级越高：

- 可表示的文件越大
- 但访问路径越长

最容易卡住的点，是题目里既给了：

- 每个地址项 `4B`
- 磁盘索引块大小 `1KB`
- 磁盘数据块大小 `1KB`

很多人会问：为什么要算 `1KB / 4B`？直接地址为什么又“存得下”？

这三个量分别在说三件不同的事：

- 地址项 `4B`：一个“地址槽位”占 4 个字节
- 索引块 `1KB`：专门拿来装很多地址项的块
- 数据块 `1KB`：真正存文件内容的块

直接地址“存得下”，是因为直接地址项里存的不是 `1KB` 的文件内容，而只是“那个 `1KB` 数据块的地址”。

所以：

- 直接地址项 = 一个门牌号
- 数据块 = 真正装货的房间

而之所以要算 `1KB / 4B`，是因为我们要知道：

- 一个索引块里最多能放多少个地址项

答案就是：

- `1KB / 4B = 1024 / 4 = 256`

这意味着：

- 一个一级索引块可以再指向 `256` 个数据块

如果题目说某文件索引节点有：

- `5` 个直接地址项
- `2` 个一级间接地址项
- `1` 个二级间接地址项

那么覆盖范围就这样算：

- 5 个直接地址项：覆盖逻辑块号 `0~4`
- 2 个一级间接地址项：每个覆盖 `256` 个逻辑块，共覆盖 `512` 个，所以范围是 `5~516`
- 二级间接地址项：从 `517` 号逻辑块开始继续往后覆盖

可以用这个图来记：

```text
inode
├─ 直接地址项 -> 数据块
├─ 直接地址项 -> 数据块
├─ 直接地址项 -> 数据块
├─ 直接地址项 -> 数据块
├─ 直接地址项 -> 数据块
├─ 一级间接地址项 -> 一级索引块 -> 数据块
├─ 一级间接地址项 -> 一级索引块 -> 数据块
└─ 二级间接地址项 -> 二级索引块 -> 一级索引块 -> 数据块
```

所以这类题不要一上来记答案，而要先做“范围判断”。

例如：

- 逻辑块号 `1`，还在 `0~4` 范围内，所以走直接地址
- 逻辑块号 `518`，已经超过 `516`，所以必须走二级间接地址

题目常考：

1. 某逻辑块号该走哪一级索引
2. 单个文件最大长度是多少

### 3.3 解题步骤

文件管理题建议这样拆：

1. 如果是路径题，先找起点。
2. 如果是位示图题，先统一单位再算块数。
3. 如果是索引节点题，先分清直接索引和间接索引覆盖范围。

### 3.4 易错点

1. 相对路径从当前目录写，不是从根写。
2. 位示图里一位对应一个物理块，不是一位对应一个字节。
3. 字长是“一个字有多少位”，不是“磁盘块大小”。
4. 索引节点保存的是地址信息，不是文件正文。

### 3.5 真题示例 1

题目：

若某文件系统的目录结构如下图所示，假设用户要访问文件 `fault.swf`，且当前工作目录为 `Swtools`，则该文件的相对路径和绝对路径分别为（B）。

- A. `\Swtools\flash\` 和 `flash\`
- B. `flash\` 和 `\Swtools\flash\`
- C. `Swtools\flash` 和 `\flash\`
- D. `\flash` 和 `\Swtools\flash\`

正确答案：`B`

详细解析：

- 当前工作目录已经给出，是 `Swtools`。
- 目标文件在 `Swtools` 下的 `flash` 子目录里。
- 所以：
  - 绝对路径必须从根开始写，应写成 `\Swtools\flash\`
  - 相对路径从当前目录开始写，只需写 `flash\`

逐项判断：

- A 把绝对路径和相对路径写反了。
- B 正确。
- C 相对路径不该把当前目录名再写一遍。
- D 相对路径不应该从根符号 `\` 开始。

这类题的关键不是背答案，而是先判断“路径起点”。

### 3.6 真题示例 2

题目：

某文件管理系统在磁盘上建立了位示图（bitmap），记录磁盘的使用情况。若计算机系统的字长为 `128` 位，磁盘的容量为 `1024GB`，物理块的大小为 `8MB`，那么该位示图的大小为（B）个字。

- A. 4096
- B. 1024
- C. 2048
- D. 9600

正确答案：`B`

详细解析：

第一步，统一单位：

- `1024GB = 1024 * 1024MB`

第二步，求物理块数：

- `1024 * 1024MB / 8MB = 131072` 块

第三步，求需要多少位：

- 每个物理块对应 1 位
- 所以总共需要 `131072` 位

第四步，求多少个字：

- 每个字 `128` 位
- `131072 / 128 = 1024`

所以答案是 `B`。

### 3.7 本节结论

- 文件管理是第四课最稳的计算板块之一。
- 路径题抓“起点”，位示图抓“单位和步骤”，索引题抓“层级覆盖范围”。

## 三、进程同步与 PV 操作（权值：15/76，约19.7%）

### 4.1 这个知识点到底在讲什么

进程同步与 PV 操作研究的是：

- 多个进程之间先后顺序怎么控制
- 多个进程如何安全共享资源

它的高频题型非常明确：

1. 前趋图翻译成 P/V 程序
2. 信号量值判断
3. PV 操作本质功能判断

### 4.2 核心概念详细讲解

#### 4.2.1 什么是同步，什么是互斥

同步：

- 强调先后顺序
- 比如 `P1` 做完，`P2` 才能开始

互斥：

- 强调同一时刻只能有一个进程使用某资源
- 比如打印机一次只能给一个进程用

很多题把这两个概念混在一起考，所以一定要区分：

- “先后约束”是同步
- “独占资源”是互斥

#### 4.2.2 P 操作和 V 操作是什么意思

P 操作可以理解为：

- 申请资源
- 或者等待某个条件成立

V 操作可以理解为：

- 释放资源
- 或者通知后继进程“可以继续了”

如果只背成“P 减 1，V 加 1”，做复杂题还是会错。

真正理解应该是：

- 谁要等，谁做 `P`
- 谁完成后通知别人，谁做 `V`

#### 4.2.3 信号量值怎么看

信号量 `S` 常见理解：

- `S > 0`：还有资源可用
- `S = 0`：没有多余资源，但还没人排队
- `S < 0`：有人在等待，绝对值常表示等待进程数

所以很多单题其实不是考代码，而是考你是否知道：

- 为什么 `-3` 表示已经有人排队

#### 4.2.4 前趋图题怎么做

前趋图表示的是执行顺序。

如果 `P1 -> P2`，意思是：

- `P1` 完成前，`P2` 不能开始

翻译成 PV 逻辑一般是：

- `P1` 完成时 `V(S1)`
- `P2` 开始前 `P(S1)`

如果一个进程有多个前驱，就说明：

- 它开始前需要做多个 `P`

如果一个进程完成后要唤醒多个后继，就说明：

- 它完成后需要做多个 `V`

### 4.3 标准做题步骤

1. 把前趋图的每条箭头单独写出来。
2. 每条箭头通常对应一个同步信号量。
3. 找共享资源，单独建立互斥信号量。
4. 谁完成前置动作，谁 `V`。
5. 谁必须等待，谁 `P`。

### 4.4 易错点

1. 一上来就套模板，不先看前趋关系。
2. 把同步信号量和互斥信号量混用。
3. 误以为 PV 操作可以保证永不死锁。错，使用不当照样死锁。
4. 只会记 `P减1 V加1`，不会结合业务关系判断。

### 4.5 真题示例 1

题目：

PV 操作是操作系统提供的具有特定功能的原语。利用 PV 操作可以（B）。

- A. 保证系统不发生死锁
- B. 实现资源的互斥使用
- C. 提高资源利用率
- D. 推迟进程使用共享资源的时间

正确答案：`B`

详细解析：

- A 为什么错：
  - PV 操作是工具，不是自动防死锁装置。
  - 信号量设计错了，照样会死锁。
- B 为什么对：
  - PV 操作最核心的作用就是控制同步与互斥。
- C 为什么错：
  - 资源利用率是否提高，要看调度与设计，不是 PV 本身直接保证。
- D 为什么错：
  - 这不是 PV 操作的核心定义。

这题考的其实就是一句话：

- PV 操作最本质的作用是“同步与互斥”。

### 4.6 真题示例 2

题目：

进程 `P1、P2、P3、P4、P5、P6` 的前趋图如下所示。假设用 PV 操作来控制这 6 个进程的同步与互斥程序，程序中的空①和空②处应分别为（C/C/D），空③和空④处应分别为（ ），空⑤和空⑥处应分别为（ ）。

问题1：

- A. `V（S1）V（S2）` 和 `P（S2）P（S3）`
- B. `V（S1）P（S2）` 和 `V（S3）P（S4）`
- C. `V（S1）V（S2）` 和 `V（S3）V（S4）`
- D. `P（S1）P（S2）` 和 `V（S2）V（S3）`

问题2：

- A. `V（S3）` 和 `V（S6）V（S7）`
- B. `V（S3）` 和 `V（S6）P（S7）`
- C. `P（S3）` 和 `V（S6）V（S7）`
- D. `P（S3）` 和 `P（S6）V（S7）`

问题3：

- A. `V（S6）` 和 `P（S7）P（S8）`
- B. `P（S8）` 和 `P（S7）P（S8）`
- C. `P（S8）` 和 `P（S7）V（S8）`
- D. `V（S8）` 和 `P（S7）P（S8）`

正确答案：

- 问题1：`C`
- 其余空也按前趋关系继续推

详细解析：

做这种题不要先看选项，要先做翻译：

1. 看图中谁是谁的前驱。
2. 每一条前驱关系都变成“前驱完成后 `V`，后继开始前 `P`”。
3. 如果一个进程完成后要唤醒两个后继，就要做两个 `V`。
4. 如果一个进程开始前要等两个前驱，就要做两个 `P`。

所以这类题的关键不是背代码，而是把图翻译成约束。

### 4.7 本节结论

- PV 题是第四课最典型的“方法题”。
- 先看关系，再写 P/V；不要倒过来。

## 四、进程调度、线程与死锁（权值：12/76，约15.8%）

### 5.1 这个知识点到底在讲什么

这一块其实是三条小主线：

1. 进程调度
2. 线程共享边界
3. 死锁与资源图

虽然看起来概念多，但题型非常集中，所以特别适合整理成固定判断模板。

另外，这一块还经常和“实时系统”一起混考。

要注意：

- 实时系统考的不是“谁更快”
- 而是“能不能在被控对象规定的时限内作出响应”

所以“一个时间片内”“一个周转时间内”这类说法都不是实时系统的核心定义。

### 5.2 核心概念详细讲解

#### 5.2.1 进程调度的目的是什么

进程调度的目标是：

- 合理分配 CPU 时间
- 提高 CPU 利用率
- 保持系统吞吐与响应能力

但考试里最稳定的标准表述是：

- 合理分配 CPU 时间，提高 CPU 利用率

为什么“提高运行速度”常常不是标准答案？

- 因为调度本身不是让 CPU 变快
- 它只是让 CPU 资源用得更合理

#### 5.2.2 什么是可剥夺调度

可剥夺调度：

- 正在运行的进程可以被更高优先级进程强制抢占 CPU

不可剥夺调度：

- 正在运行的进程不主动释放 CPU 前，别的进程不能直接抢走

所以题目只要说：

- “系统强行撤下 P1，让 P2 运行”

就是可剥夺调度。

还有一类状态题很容易和调度题一起考：

- 进程释放了某个外设
- 但不代表它就自动让出 CPU

例如单处理机系统中：

- `P1` 正在运行
- `P2` 在就绪队列
- `P4` 等待扫描仪

如果此时 `P1` 释放了扫描仪，最常见的正确判断是：

- `P1` 仍然是运行态，因为它释放的是扫描仪，不是 CPU
- `P2` 仍然是就绪态，因为 CPU 还在给 `P1`
- `P4` 从等待态转成就绪态，因为它等的扫描仪现在可用了

所以状态题一定要把两件事分开：

1. 谁在占用 CPU
2. 谁在等待某个外设或事件

#### 5.2.3 线程和进程到底怎么区分

可以这样记：

- 进程是资源分配的基本单位
- 线程是 CPU 调度的基本单位

线程共享的通常是进程级资源，比如：

- 代码段
- 打开的文件
- 全局变量

线程不共享的通常是自己的运行现场，比如：

- 栈
- 栈指针
- 部分寄存器上下文

所以“栈指针能不能共享”是线程题最常见的陷阱点。

#### 5.2.4 死锁怎么理解

死锁可以简单理解成：

- 每个进程都在等别人释放资源
- 谁也走不下去

考试里最常见的死锁题有两类：

1. 求最少资源数，保证不死锁
2. 看资源图能不能化简

资源图题的本质是：

- 如果还能找到能继续执行并释放资源的进程，就继续化简
- 如果所有进程都卡住，图就不可化简，说明存在死锁或死锁风险

如果题目是“有 `n` 个进程，每个进程最多需要 `m` 个同类资源，至少多少资源才能保证不死锁”，最常用结论是：

- `n * (m - 1) + 1`

直观理解是：

- 先让每个进程都差 `1` 个资源拿不到，即一共先分出去 `n * (m - 1)` 个
- 这时如果系统手里还剩 `1` 个资源，就总能让某一个进程先凑够资源执行完
- 它一执行完，就会释放资源，系统就能继续推进，不会僵死

### 5.3 解题步骤

1. 调度题：先看有没有“抢占”。
2. 线程题：先分“进程级共享资源”和“线程级私有现场”。
3. 死锁题：先看资源总数，再看是否能释放，再看图是否能化简。

### 5.4 易错点

1. 把调度目标写成“提高速度”。
2. 把线程私有栈当成共享资源。
3. 看死锁题时只看表面，不去尝试化简资源图。

### 5.5 真题示例 1

题目：

在操作系统中，进程调度的主要目的是（A）。

- A. 合理分配CPU时间，提高CPU利用率
- B. 提高计算机的运行速度
- C. 减少进程的等待时间
- D. 增加系统的吞吐量

正确答案：`A`

详细解析：

- A 为什么对：
  - 这是最标准、最核心的调度目标表述。
- B 为什么错：
  - 调度不改变 CPU 的硬件速度。
- C 为什么错：
  - 减少等待时间可能是结果之一，但不是教材里最标准的主定义。
- D 为什么不作为最佳项：
  - 吞吐量确实相关，但题目问“主要目的”，标准答案仍是 CPU 利用率这一表达。

### 5.6 真题示例 2

题目：

在支持多线程的操作系统中，假设进程 `P` 创建了 `t1、t2、t3` 线程，那么（C）。

- A. 该进程的代码段不能被 `t1、t2、t3` 共享
- B. 该进程的全局变量只能被共享
- C. 该进程中 `t1、t2、t3` 的栈指针不能被共享
- D. 该进程中 `t1` 的栈指针可以被 `t2、t3` 共享

正确答案：`C`

详细解析：

- A 错：
  - 同一进程中的多个线程本来就共享代码段。
- B 表述不完整且误导：
  - 全局变量确实可共享，但题目核心在“哪个不能共享”。
- C 对：
  - 栈指针属于线程自己的运行现场，不能共享。
- D 错：
  - 栈指针共享会导致线程运行现场混乱。

### 5.7 本节结论

- 这一块的核心不是背长定义，而是建立边界感。
- 调度看“是否抢占”，线程看“哪些私有”，死锁看“能否化简”。

## 五、存储管理（权值：11/76，约14.5%）

### 6.1 这个知识点到底在讲什么

存储管理研究的是：

- 程序地址怎么映射到物理内存
- 内存不够时怎么扩展和替换

第四课里最常见的是：

1. 分页地址结构
2. 页表地址变换
3. 页面置换
4. 段页式地址划分

### 6.2 核心概念详细讲解

#### 6.2.1 什么是分页

分页就是把：

- 逻辑地址空间分成大小相等的“页”
- 物理内存分成大小相等的“块”

页和块大小相同，这样页就可以装进任意一个块里。

逻辑地址通常分成两部分：

1. 页号
2. 页内偏移

其中：

- 页号决定访问第几页
- 页内偏移决定访问该页中的第几个字节

#### 6.2.2 页内偏移和页面大小的关系

如果页内偏移占 `n` 位，那么：

- 页面大小 = `2^n`

例如偏移是 `12` 位：

- 页面大小就是 `2^12 = 4096B = 4KB`

#### 6.2.3 页号和最大页数的关系

如果页号占 `m` 位，那么：

- 最大页数 = `2^m`

所以分页地址题常常就是在考：

- 偏移位数看页面大小
- 页号位数看页数量

#### 6.2.4 页表地址变换怎么做

页表记录的是：

- 某逻辑页号对应哪个物理块号

做地址变换时：

1. 先从逻辑地址里拆出页号和页内偏移
2. 查页表，得到物理块号
3. 把物理块号和页内偏移拼起来
4. 得到物理地址

#### 6.2.5 页面置换题看什么

页面置换的题目里，常会给你：

- 状态位
- 访问位
- 修改位

做题时要先做两件事：

1. 先排除根本不在内存的页
2. 再在“在内存的候选页”里按规则比较

这一点非常重要，因为很多人还没筛候选页就开始选答案。

#### 6.2.6 什么是段页式

段页式不是“先分页再分段”，而是：

- 先把程序按逻辑意义分成段
- 每一段内部再分页

所以段页式题一般会给一张地址位图，问你：

- 段号多少位
- 页号多少位
- 页内偏移多少位

### 6.3 解题步骤

1. 先看题目是分页、段页式还是页面置换。
2. 如果是地址结构题，先数位数。
3. 如果是地址变换题，先拆页号和偏移，再查页表。
4. 如果是页面置换题，先找在内存中的候选页。

### 6.4 易错点

1. 把页号位数和偏移位数搞反。
2. 页面置换时没先筛掉不在内存的页。
3. 段页式看成单纯分页。

### 6.5 真题示例 1

题目：

在分页管理系统中，逻辑地址由页面编号和偏移值构成。如下所示的逻辑地址表示，（B）。

- A. 页面大小为2K，每个页面最大为8M
- B. 页面大小为4K，每个页面最大为1M
- C. 页面大小为8K，每个页面最大为2M
- D. 页面大小为1K，每个页面最大为16M

正确答案：`B`

详细解析：

题目本质是在考两句话：

1. 页内偏移位数决定页面大小
2. 页号位数决定最大页数

如果偏移占 `12` 位：

- 页面大小 = `2^12 = 4KB`

如果页号占 `20` 位：

- 最大页数 = `2^20 = 1M`

所以答案是 `B`。

### 6.6 真题示例 2

题目：

进程 `P` 有 `5` 个页面，页号为 `0~4`，页面变换表及状态位、访问位和修改位的含义如下图所示。若系统给进程 `P` 分配了 `3` 个存储块，当访问的页面 `3` 不在内存时，应该淘汰表中页号为（A）的页面。

- A. 0
- B. 1
- C. 2
- D. 4

正确答案：`A`

详细解析：

这题最关键的是不能直接在所有页里选。

第一步，先找在内存中的页面：

- 只有在内存中的页才可能被淘汰

第二步，再按题目给的访问位、修改位规则比较：

- 先看访问位
- 再看修改位

最后选出代价最小的候选页。

所以页面置换题真正考的是：

- 候选页筛选
- 位信息比较顺序

### 6.7 本节结论

- 存储管理题看起来像算题，实质上是规则题。
- 只要步骤固定，正确率会很高。

## 六、作业管理与 SPOOLing（权值：1/76，约1.3%）

### 7.1 这个知识点到底在讲什么

作业管理关注的是：

- 作业如何进入系统
- 作业如何被调度执行
- 用户如何与系统交互

但在本地客观题里，这一块独立考得不多，最稳定出现的是 `SPOOLing`。

### 7.2 核心概念详细讲解

#### 7.2.1 什么是 SPOOLing

SPOOLing 可以看成一种“排队 + 缓冲 + 虚拟共享”机制。

典型场景是打印机：

- 打印机本来是独占设备
- 一次只能给一个进程直接使用

有了 SPOOLing 后：

- 多个进程先把输出内容放进磁盘缓冲区
- 系统按队列顺序送给打印机
- 对用户来说，好像打印机可以“被多人共享”

所以 SPOOLing 的主目的不是：

- 提高内存速度
- 提高 CPU 频率
- 直接提高并发度

而是：

- 将独占设备虚拟成共享设备

#### 7.2.2 为什么它常和设备管理一起考

因为它本质上还是在处理“慢速外设怎么被多个进程使用”的问题。

所以你可以把它当成设备管理的延伸点来记，不需要单独割裂背诵。

### 7.3 易错点

1. 把“提高并发度”当成主目的。
2. 把“提高 CPU 利用率”当成标准答案。
3. 忘了它最典型的应用对象就是打印机这类独占设备。

### 7.4 真题示例

题目：

使用 SPOOLing 技术的主要目的是（A）。

- A. 将独占设备虚拟为共享设备
- B. 提高进程并发度
- C. 加快内存访问速度
- D. 提高CPU利用率

正确答案：`A`

详细解析：

- A 为什么对：
  - SPOOLing 的核心就是让独占设备在逻辑上表现为共享设备。
- B 为什么错：
  - 并发度可能间接提升，但这不是最本质定义。
- C 为什么错：
  - 它和内存访问速度没有直接关系。
- D 为什么错：
  - CPU 利用率可能受系统整体影响，但这也不是主目标。

记忆时只抓一句母句：

- `SPOOLing = 独占设备虚拟为共享设备`

### 7.5 本节结论

- 这块权值最轻，但概念很典型。
- 最稳做法是把它并入设备管理一起复习。
