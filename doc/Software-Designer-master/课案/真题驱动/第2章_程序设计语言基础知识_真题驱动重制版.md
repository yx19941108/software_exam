## 一、编译程序工作过程（权值：6/20，约30.0%）

### 2.1 这个知识点到底在讲什么

这一块考的是：源程序从“人能写的代码”变成“机器能执行的东西”，中间要经历哪些阶段。

### 2.2 核心概念详细讲解

编译程序的核心处理链路要记成固定顺序：

1. 源程序字符流
2. 词法分析
3. 记号流
4. 语法分析
5. 语义分析
6. 中间代码生成
7. 代码优化
8. 目标代码生成

`词法分析`：

- 输入是字符流
- 输出是 token / 记号流
- 负责识别关键字、标识符、常量、运算符

`语法分析`：

- 检查结构是否符合文法
- 关注“句子长得对不对”

`语义分析`：

- 检查“意思对不对”
- 例如类型匹配、先定义后使用

`编译程序`：

- 先翻译再执行
- 运行时通常不参与控制

`解释程序`：

- 一边翻译一边执行
- 一般不生成可独立运行的目标程序

`汇编程序`：

- 把汇编语言翻译成机器语言或目标代码

### 2.3 易错点

1. 把词法分析和语法分析混在一起。
2. 把“解释程序”说成一定会生成独立目标程序。
3. 记不住“解释器参与运行控制，编译器一般不参与”。

### 2.4 真题示例

题目：

以下关于高级程序设计语言实现的编译和解释方式的叙述中，正确的是（A）。

- A. 编译程序不参与用户程序的运行控制，而解释程序则参与
- B. 编译程序可以用高级语言编写，而解释程序只能用汇编语言编写
- C. 编译方式处理源程序时不进行语义分析
- D. 解释程序一定比编译程序执行得快

正确答案：`A`

详细解析：

- A 对：编译器完成翻译后，目标程序独立运行；解释器通常在执行过程中参与控制。
- B 错：两者都可以用高级语言实现。
- C 错：编译程序当然包含语义分析阶段。
- D 错：解释方式通常不比编译后机器码执行快。

## 二、表达式、后缀式与栈求值（权值：5/20，约25.0%）

### 3.1 这个知识点到底在讲什么

这部分高频考的是：

- 中缀式和后缀式转换
- 为什么后缀式适合用栈求值
- 栈最大深度怎么判断

### 3.2 核心概念详细讲解

`中缀表达式`：

- 运算符写在操作数中间
- 例如 `a+b*c`

`后缀表达式`：

- 运算符写在操作数后面
- 例如 `a b c * +`

后缀式不依赖括号，因为：

- 扫描到运算符时，最近两个操作数就是它的运算对象

利用后缀式求值时：

1. 遇到操作数就入栈
2. 遇到运算符就弹出两个操作数计算
3. 结果再压回栈

### 3.3 易错点

1. 只会看答案，不会按“先算谁”来转换中缀式。
2. 不会用栈来模拟后缀式求值。

### 3.4 真题示例

题目：

表达式采用逆波兰式表示时，利用（A）进行求值。

- A. 栈
- B. 队列
- C. 符号表
- D. 链表

正确答案：`A`

详细解析：

- 逆波兰式就是后缀式。
- 后缀式求值的天然数据结构就是栈，因为它正好满足“最近进入的操作数最先参与运算”的规则。
- 所以 A 对。

## 三、Python 基础语法与容器（权值：5/20，约25.0%）

### 4.1 这个知识点到底在讲什么

本地题库里的程序语言题，近几年已经明显混进不少 Python 基础语法和容器题。

高频点主要是：

- `list`
- `tuple`
- `dict`
- `set`
- `append`
- `extend`
- `+`
- `*`

### 4.2 核心概念详细讲解

`list`：

- 有序
- 可重复
- 可变

`tuple`：

- 有序
- 可重复
- 不可变

`dict`：

- 键值映射
- 按键访问

`set`：

- 无序
- 元素不重复

Python 容器常考运算：

- `list + list` 是拼接
- `list * n` 是重复
- `tuple + tuple` 也是拼接
- `tuple * n` 也是重复

### 4.3 易错点

1. 把 `list + list` 误当成逐元素相加。
2. 把 `tuple` 当成可修改容器。
3. 把 `dict` 当成顺序表用下标访问。

### 4.4 真题示例

题目：

执行以下 Python 语句之后，列表 `y` 为（B）。

```python
x=[1,2,3]
y=x+[4,5,6]
```

- A. 出错
- B. `[1,2,3,4,5,6]`
- C. `[5,7,9]`
- D. `[1,2,3,[4,5,6]]`

正确答案：`B`

详细解析：

- Python 中，`list + list` 表示拼接，不是逐元素求和。
- 所以 `y` 的结果是把后一个列表直接接到前一个后面。
- 因此 B 对。
- C 是很多人按数学向量加法去想才会选出来的错误项。
- D 更像 `append([4,5,6])` 的效果，不是 `+` 的效果。

## 四、参数传递与语言特征（权值：4/20，约20.0%）

### 5.1 这个知识点到底在讲什么

这一块考的是：

- 传值调用
- 引用调用
- 某些语言的运行特征

### 5.2 核心概念详细讲解

`传值调用`：

- 形参拿到的是实参值的副本
- 形参怎么改，通常不影响原实参

`引用调用`：

- 形参与实参绑定到同一对象或同一存储语义
- 形参变化可反映到实参

做题时一定先看清：

1. 哪个参数是传值
2. 哪个参数是传引用
3. 函数结束后被问的是哪个变量

### 5.3 真题示例

题目：

程序运行过程中常使用参数在函数（过程）间传递消息，引用调用传递的是实参的（A）。

- A. 地址
- B. 类型
- C. 名称
- D. 值

正确答案：`A`

详细解析：

- 传值调用传递的是“值的副本”。
- 引用调用传递的是“实参所在存储单元的地址或地址引用”。
- 因此 A 正确。
- B、C、D 都没有抓住“引用调用为什么会影响原变量”的根本原因。

所以参数传递题最该抓住的一句话是：

- `传值传的是值，引用传的是地址。`
