# 第6章 结构化开发方法

- 本课只基于本地上午客观题生成。
- 采用 `18` 题代表性题池做权值统计。
- 权值定义：`该知识点题数 / 18`。

## 1. 本地题池与知识点权值

1. 结构化分析基础与 DFD：`6/18`
2. 数据字典与加工规格说明：`5/18`
3. 结构化设计与模块结构：`4/18`
4. 原型、WebApp 与用户界面：`3/18`

## 一、结构化分析基础与 DFD（权值：6/18，约33.3%）

### 2.1 这个知识点到底在讲什么

结构化分析强调用一套清晰、可分层的方法，把“系统怎么处理数据”描述出来。

它的核心思想是：

- 自顶向下
- 逐步分解
- 先整体后局部

这部分高频考的是 `DFD`，也就是数据流图。

### 2.2 核心概念详细讲解

数据流图不关心界面长什么样，也不关心代码怎么写。它只关心：

- 数据从哪里来
- 到哪里去
- 在系统中经过哪些加工

DFD 的四个核心元素是：

1. 外部实体
2. 数据流
3. 加工
4. 数据存储

其中：

- `外部实体` 是系统外部的人、组织或外部系统
- `加工` 是系统内部对数据做变换的地方
- `数据流` 是数据的流动
- `数据存储` 是系统内部长期保存数据的地方

### 2.3 易错点

1. 把外部系统误判成加工。
2. 把 DFD 当成“控制流图”，其实它描述的是数据流，不是控制流。
3. 忘记 DFD 的顶层图一般只有一个总加工框。

### 2.4 真题示例

题目：

结构化分析方法的基本思想是（B）。

- A. 自底向上逐步分解
- B. 自顶向下逐步分解
- C. 自底向上逐步抽象
- D. 自顶向下逐步抽象

正确答案：`B`

详细解析：

- 结构化分析的核心口号就是：`自顶向下、逐步分解`。
- A、C 把方向写反了。
- D 虽然有“自顶向下”，但强调的是抽象，不是这套方法的标准表述。

## 二、数据字典与加工规格说明（权值：5/18，约27.8%）

### 3.1 这个知识点到底在讲什么

DFD 只是画框和箭头，不够细，所以必须配套两样东西：

1. 数据字典
2. 加工规格说明

### 3.2 核心概念详细讲解

`数据字典` 是对 DFD 中数据相关元素的说明书。

它通常记录：

- 数据项
- 数据流
- 数据存储
- 基本加工

它不等于：

- 外部实体总表
- 代码实现细节

`加工规格说明` 用来说明一个基本加工怎么把输入变成输出。

常用表示方法包括：

- 结构化语言
- 决策表
- 决策树

如果一个加工要根据多个条件组合决定动作，最常见答案就是：

- `判定表` 或 `决策表`

### 3.3 真题示例

题目：

数据字典是结构化分析的一个重要输出。数据字典的条目不包括（A）。

- A. 外部实体
- B. 数据流
- C. 数据项
- D. 基本加工

正确答案：`A`

详细解析：

- 数据字典核心是解释“系统内部数据相关元素”。
- 数据流、数据项、基本加工都属于常见条目。
- 外部实体通常在 DFD 中标识，但不是数据字典的核心条目类型。

## 三、结构化设计与模块结构（权值：4/18，约22.2%）

### 4.1 这个知识点到底在讲什么

结构化设计是在分析完成后，把系统落实成模块结构。

它常考：

- 结构图由什么构成
- 模块设计原则
- 模块独立性

### 4.2 核心概念详细讲解

结构图主要描述：

- 模块
- 调用关系
- 数据传递
- 控制信息

结构化设计追求：

- 高内聚
- 低耦合
- 模块规模适中
- 控制域和作用域关系合理

### 4.3 真题示例

题目：

结构化设计方法中使用结构图来描述构成软件系统的模块以及这些模块之间的调用关系。结构图的基本成分不包括（D）。

- A. 模块
- B. 调用
- C. 数据
- D. 控制

正确答案：`D`

详细解析：

- 结构图里强调的是模块、调用以及数据/控制信息的传递符号。
- 但选项中的“控制”不是标准成分表述，标准说法应是“控制信息”。
- 所以这类题常用来卡你对术语精确表达的掌握程度。

## 四、原型、WebApp 与用户界面（权值：3/18，约16.7%）

### 5.1 这个知识点到底在讲什么

结构化开发方法并不只会考 DFD 和结构图，还会带一点：

- 原型在需求澄清中的作用
- WebApp 的分析与设计
- 用户界面设计

### 5.2 核心概念详细讲解

`原型` 最适合做两类事：

- 帮助用户把需求说清楚
- 帮助设计者验证方案是否合理

它不擅长：

- 指导代码优化
- 替代完整工程设计

### 5.3 真题示例

题目：

以下关于系统原型的叙述中，不正确的是（C）。

- A. 可以帮助导出系统需求并验证需求的有效性
- B. 可以用来探索特殊的软件解决方案
- C. 可以用来指导代码优化
- D. 可以用来支持用户界面设计

正确答案：`C`

详细解析：

- A 对：原型最大价值之一就是澄清需求。
- B 对：探索型原型本来就是为了验证解决思路。
- C 错：原型不负责代码级优化。
- D 对：界面原型是最常见用途之一。
