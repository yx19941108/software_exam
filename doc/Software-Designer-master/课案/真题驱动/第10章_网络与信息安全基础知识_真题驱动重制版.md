## 2. 网络设备、分层模型与地址机制（权值：4/19）

### 2.1 先把“层”当作职责边界

考试中的网络分层不是让你背名词，而是让你判断“这个设备或协议主要处理什么对象”。

- 物理层处理比特流和电气/光信号，典型设备是中继器、集线器。
- 数据链路层处理同一链路或同一局域网内的帧，典型设备是交换机、网桥，核心地址是 MAC 地址。
- 网络层处理跨网络转发，典型设备是路由器，核心地址是 IP 地址。
- 传输层处理端到端进程通信，典型协议是 TCP、UDP，核心标识是端口号。
- 应用层面向具体应用服务，例如 HTTP、FTP、SMTP、POP3、DNS、SNMP。

做题时不要从设备名字硬背，要问三件事：

1. 它处理的是信号、帧、IP 包，还是应用数据？
2. 它根据 MAC 地址、IP 地址，还是端口号做判断？
3. 它作用在一个局域网内部，还是跨网络转发？

补充：VLAN tag 也放在这个判断框架里。VLAN 是在交换网络里做“逻辑分组”的技术，常见实现依赖交换机和帧标记，所以考试问 VLAN tag 在 OSI 哪一层实现，答案通常指向数据链路层。不要因为 VLAN 能隔离广播域，就误判成网络层；它的标记和转发基础仍然是二层帧。

数据链路层还常被考成一句话：“在物理线路上提供可靠的数据传输”。这里的可靠不是端到端 TCP 可靠传输，而是相邻节点之间以帧为单位做差错控制、流量控制和链路管理。物理层只提供比特传输通道，不能因为题干出现“物理线路”就选物理层；题干说“让有差错的物理线路变成较可靠的数据链路”，应选数据链路层。

### 2.2 IP 地址、MAC 地址、ARP 的关系

IP 地址用于网络层寻址，体现网络拓扑；MAC 地址用于数据链路层寻址，通常由网卡厂商分配。真正发到同一链路上时，网卡要知道对方 MAC 地址，因此需要 ARP。

IP 地址和 MAC 地址的区别还常以“谁唯一、谁按什么分配”来考：

- IP 地址可以是 32 位 IPv4，也可以是 128 位 IPv6；它用于网络层，分配通常跟网络拓扑、网段、运营商或管理员配置有关。
- MAC 地址通常是 48 位；它用于数据链路层，通常固化或分配给网卡硬件，考试语境下把它看成设备的唯一硬件地址。
- 因此，“IP 地址具有唯一性，MAC 地址不具有唯一性”是反着说。更稳的判断是：IP 地址可能变化、可能由 DHCP 临时分配；MAC 地址更接近设备硬件标识。

ARP 的过程可以这样记：

- 已知目标 IP，不知道目标 MAC。
- ARP Request 要问“谁是这个 IP”，因为不知道具体问谁，所以用广播发。
- 拥有该 IP 的主机返回 ARP Response，已经知道请求者是谁，所以用单播回。
- RARP 是反向地址解析，方向相反：由 MAC 地址找 IP 地址。

### 2.3 代表真题 1：设备所在层

来源：`2024上半年选择题.md` 第1题

题干：在计算机网络协议5层体系结构中，（B）工作在数据链路层。

- A. 路由器
- B. 以太网交换机
- C. 防火墙
- D. 集线器

答案：B

解析：交换机按 MAC 地址转发帧，属于数据链路层。路由器按 IP 地址转发，属于网络层；集线器只放大和转发物理信号，属于物理层；防火墙要看类型，包过滤防火墙常在网络层检查 IP 包，应用级防火墙可看应用层数据，不能简单归到数据链路层。

### 2.4 代表真题 2：ARP 的广播与单播

来源：`2021下半年选择题.md` 第53题

题干：ARP 报文分为 ARP Request 和 ARP Response，其中 ARP Request 采用（A/D）进行传送，ARP Response 采用（ ）进行传送。

问题1：

- A. 广播
- B. 组播
- C. 多播
- D. 单播

问题2：

- A. 组播
- B. 广播
- C. 多播
- D. 单播

答案：A、D

解析：ARP Request 是“我不知道谁拥有这个 IP”，所以要广播；ARP Response 是“我就是这个 IP，对应 MAC 是我”，已经知道请求者，因此单播回复。不要把“请求/响应”机械对应成 TCP 连接，这里考的是局域网地址解析过程。

## 3. IP 地址与发送范围（权值：4/19 中的地址部分）

IP 地址题常考“发给谁”。

- 单播地址：发给一台特定主机。
- 广播地址：发给同一广播范围内所有主机。
- 组播地址：发给加入某个组播组的一组主机。
- 网络地址：标识一个网络本身，不用于表示具体主机。

做地址类选择题时，先看题干关键词：

- “单个设备”对应单播。
- “所有设备”对应广播。
- “一组特定设备”对应组播。
- “标识网络”对应网络地址。

地址长度题属于硬记点：IPv4 是 32 位，IPv6 是 128 位。IPv6 地址空间相对 IPv4 扩大为 `2^128 / 2^32 = 2^96` 倍。题目只问“IPv6 地址长度”时直接选 128 bit。

### 3.1 代表真题：组播地址

来源：`2025上半年选择题.md` 第6题

题干：（C）用于在网络中向一组特定的设备发送数据包。

- A. 网络地址
- B. 广播地址
- C. 组播地址
- D. 单播地址

答案：C

解析：题干中的“一组特定设备”是组播的典型提示。广播是所有设备，单播是一个设备，网络地址是网络本身。

## 4. TCP/IP 协议、Internet 服务与端口（权值：4/19）

### 4.1 先分清协议层次

TCP/IP 题常把应用层协议、传输层协议、网络层协议混在选项里。

- HTTP：网页访问的应用层协议，默认端口 80。
- HTTPS：HTTP 加 TLS/SSL 安全层，默认端口 443。
- FTP：文件传输协议，常见控制端口 21。
- SMTP：发送邮件，常见端口 25。
- POP3：接收邮件，基于 TCP，默认端口 110。
- DNS：域名解析，把域名转成 IP 地址，常用端口 53。
- SNMP：网络管理协议，传输层通常使用 UDP。
- TCP：传输层可靠连接协议。
- UDP：传输层无连接协议，开销小，不保证可靠交付。
- ICMP：网络层控制报文，ping 常用它测试连通性。

判断题目时先问：题干说的是“网页、邮件、域名、文件、网络管理”这种服务，还是“可靠连接、无连接、端口”这种传输能力。

补充两个容易丢分的协议细节：

- POP3 是接收邮件协议，不是发送邮件协议；客户端收邮件时与 POP3 服务器通过 TCP 建立连接，默认端口是 110。SMTP 才是发送邮件常考协议。
- SNMP 是网络管理协议，常用 UDP，而不是 TCP。它追求管理报文开销小，考试常问“SNMP 的传输层协议是什么”，直接选 UDP。
- SFTP 是 SSH File Transfer Protocol，强调安全文件传输；FTP 是普通文件传输，TFTP 更简单且不可靠，ICMP 是控制报文协议，不负责文件传输。
- TCP 序号的单位是字节。TCP 用序号保证字节流按顺序交付、发现丢失并触发重传；它不是按比特、赫兹或整个报文编号。

路由协议也会被放在第10章考：

- 内部网关协议 IGP 用在一个自治系统内部，例如 RIP、OSPF。
- 外部网关协议 EGP 用在自治系统之间，典型代表是 BGP。
- UDP 是传输层协议，不是路由协议。看到“自治系统之间”优先想 BGP。
- 路由协议的任务是让路由器之间交换路由信息、学习网络拓扑、生成或更新路由表，最终帮助路由器做路径选择。
- “允许数据包在主机之间传送信息”描述的是网络通信/转发的泛化结果，不是路由协议本身的定义。题目问“路由协议叙述错误”时，这类把路由协议说成主机间传送信息协议的选项通常是错项。

### 4.2 Web 访问过程

浏览器访问网页的基本过程是：

1. 用户输入 URL。
2. 浏览器做 DNS 解析，得到服务器 IP 地址。
3. 浏览器按目标 IP 和端口建立 TCP 连接。
4. 浏览器发送 HTTP 请求。
5. 服务器返回网页数据。
6. 连接关闭。
7. 浏览器解析并显示页面。

如果浏览器打不开网页，但即时聊天软件正常，优先怀疑 DNS，因为聊天软件可能直接连已知 IP 或自带解析链路，而网页访问通常强依赖域名解析。

### 4.3 代表真题 1：Web 默认端口

来源：`2024下半年选择题.md` 第5题

题干：访问 Web 服务器默认使用的端口号是（C）。

- A. 53
- B. 110
- C. 80
- D. 23

答案：C

解析：Web 默认是 HTTP，HTTP 默认端口是 80。53 是 DNS，110 是 POP3，23 是 Telnet。端口题不要只背数字，还要连到服务名称。

### 4.4 代表真题 2：HTTP 请求顺序

来源：`2019上半年选择题.md` 第57题

题干：下面是 HTTP 的一次请求过程，正确的顺序是（A）。

①浏览器向 DNS 服务器发出域名解析请求并获得结果  
②在浏览器中输入 URL，并按下回车键  
③服务器将网页数据发送给浏览器  
④根据目的 IP 地址和端口号，与服务器建立 TCP 连接  
⑤浏览器向服务器发送数据请求  
⑥浏览器解析收到的数据并显示  
⑦通信完成，断开 TCP 连接

- A. ②①④⑤③⑦⑥
- B. ②①⑤④③⑦⑥
- C. ②①④⑤③⑥⑦
- D. ②①④③⑤⑦⑥

答案：A

解析：先输入 URL，再 DNS 解析；拿到 IP 后才能建立 TCP 连接；连接后才能发 HTTP 请求；服务器返回数据；连接关闭；浏览器解析显示。常见错法是把 HTTP 请求放在 TCP 连接前，或者把显示页面放在连接关闭前。

## 5. 信息安全与密码机制（权值：5/19）

### 5.1 先分清四个目标

信息安全题最常见的错误是把“加密、摘要、认证、签名”混成一件事。

- 保密性：不让别人看懂内容，通常靠加密。
- 完整性：证明内容没有被改，常用报文摘要或哈希。
- 身份认证：确认对方是谁，常用证书、口令、Kerberos 等机制。
- 不可否认性：发送者事后不能否认发送过，常用数字签名。

### 5.2 对称加密、非对称加密、摘要、数字签名

对称加密只有一把共享密钥，加密和解密用同一把钥匙，速度快，适合大量数据。典型算法有 DES、3DES、AES。

非对称加密有一对密钥：公钥和私钥。公钥可以公开，私钥必须保密。典型算法有 RSA、ECC、DSA。它适合密钥交换、身份认证、数字签名，但直接加密大量数据效率低。

报文摘要不是“把内容加密”。摘要算法把任意长度数据计算成固定长度摘要值，主要用于发现内容是否被篡改。常见关键词是 Hash、SHA、MD5。

数字签名通常是“先做摘要，再用发送者私钥处理摘要”。接收方用发送者公钥验证。它同时服务于完整性、身份来源确认和不可否认性。

算法分类要按“密钥使用方式”判断：

- DES、3DES、AES 是对称加密算法，加密和解密使用同一把密钥。
- RSA、ECC、DSA 属于公开密钥/非对称体系。RSA 和 ECC 常用于公钥加密、密钥交换或签名场景；DSA 主要用于数字签名。
- MD5、SHA 这类 Hash/摘要算法不是用来保密传输内容，而是把报文压缩成摘要，用于校验完整性。看到“防篡改”优先想摘要；看到“防窃听”才优先想加密；看到“防抵赖”优先想数字签名。
- MD5 的摘要长度是 128 位。题目如果问“MD5 是什么算法、结果多少位”，要答“摘要算法、128 位”。

数字签名的密钥方向要固定记：

- 发送方签名：用发送方自己的私钥。
- 接收方验证：用发送方的公钥。
- 这能验证报文完整性、发送方身份，并帮助防抵赖；不要用接收方的公钥或私钥来解释签名验证。

### 5.3 HTTPS、SSL/TLS 与证书

HTTPS 可以理解为 HTTP 加安全通道。实际通信中常见做法是：

1. 浏览器验证服务器证书，判断服务器是否可信。
2. 双方协商会话密钥。
3. 后续 HTTP 内容主要用会话密钥做对称加密。

为什么不一直用公钥加密？因为非对称加密速度慢，适合协商密钥，不适合大量数据传输。证书被撤销时，核心后果不是“服务器不能加密”，而是“客户端不能再信任服务器身份”。

考试里 HTTPS 还常考两个硬记点：

- HTTPS 基于 SSL/TLS 提供安全传输，不是基于 RSA、DES 或 SSH。RSA、DES 是算法；SSH 是远程安全登录协议。
- HTTP 默认端口是 80，HTTPS 默认端口是 443。
- IPsec、SFTP、HTTPS 都属于安全相关协议；SNMP 是网络管理协议，本身不应直接归为安全协议。题目问“不属于安全协议”，SNMP 常是答案。

### 5.4 代表真题：HTTPS 中的会话密钥与证书

来源：`2023上半年选择题.md` 第8题

题干：PKI 体系中，由 SSL/TSL 实现 HTTPS 应用。浏览器和服务器之间用于加密 HTTP 消息的方式是（D/C）。如果服务器证书被撤销，那么所产生的后果是（ ）。

问题1：

- A. 对方公钥+公钥加密
- B. 本方公钥+公钥加密
- C. 会话密钥+公钥加密
- D. 会话密钥+对称加密

问题2：

- A. 服务器不能执行加解密
- B. 服务器不能执行签名
- C. 客户端无法再信任服务器
- D. 客户端无法发送加密信息给服务器

答案：D、C

解析：HTTPS 建立安全连接后，真正承载大量 HTTP 数据的是会话密钥加对称加密；证书负责证明服务器身份。证书被撤销，客户端无法再信任服务器，而不是服务器突然失去计算加解密的能力。

## 6. 网络安全设备、区域与访问控制（权值：3/19）

### 6.1 防火墙不是万能安全设备

防火墙的核心是按规则控制流量。不同类型防火墙检查的层次不同：

- 包过滤防火墙：主要在网络层检查 IP 包，也会看协议、源/目的 IP、源/目的端口等。
- 应用级防火墙：能检查应用层数据，粒度更细。
- Web 应用防火墙：重点防 Web 攻击，例如 SQL 注入、XSS 等。

ACL 是访问控制列表，用规则决定允许或拒绝哪些流量。题干说“阻止外部未授权用户访问内部网络”，常见答案是 ACL。

入侵检测系统 IDS 的重点是发现、告警，不等于直接阻断所有攻击。入侵防御系统 IPS 才更强调在线阻断。

SQL 注入防护要区分“预防/阻断”和“发现/告警”：

- 对用户输入做过滤、参数化查询、部署 Web 应用防火墙、修复漏洞，都属于更直接的防护思路。
- IDS 主要做检测和告警，不能默认等同于阻断 SQL 注入。题目问“不能够有效防御 SQL 注入”的时候，IDS 往往是陷阱项。

防火墙、IDS、漏洞扫描的边界要分清：

- 防火墙常见功能：控制进出网络的数据包和数据流向，记录日志和审计，隐藏内部 IP 与网络结构细节。
- 防火墙不等于漏洞扫描器。题目问“不属于防火墙功能特性”，`提供漏洞扫描功能`通常是错误项。
- IDS 的常见检测技术包括专家系统、模型检测、简单匹配等；漏洞扫描是发现系统弱点的安全评估手段，不属于入侵检测技术本身。

### 6.2 DMZ 的保护等级

DMZ 是非军事区，常放 Web、邮件等需要被外部访问的服务器。它比外网更受保护，但不能像内网一样高信任。

保护程度从低到高：

外网 < DMZ < 内网

### 6.3 代表真题：DMZ 保护等级

来源：`2021下半年选择题.md` 第11题

题干：防火墙通常分为内网、外网和 DMZ 三个区域，按照受保护程度，从低到高正确的排列次序为（B）。

- A. 内网、外网和 DMZ
- B. 外网、DMZ 和内网
- C. DMZ、内网和外网
- D. 内网、DMZ 和外网

答案：B

解析：外网最不可信，内网最可信，DMZ 位于两者之间，用来放置需要对外服务但不能直接暴露内网的服务器。

## 7. 网络攻击与恶意代码（权值：2/19）

### 7.1 恶意代码和攻击方式要分开

病毒、蠕虫、木马偏“恶意代码类型”；DoS、DDoS、SQL 注入、XSS 偏“攻击方式”。考试常让你判断某项“不属于计算机病毒”。

- 病毒：通常依附在程序或文件上，运行时传播或破坏。
- 蠕虫：更强调自我复制和网络传播。
- 木马：伪装成正常程序，诱导运行后执行恶意行为。
- DoS/DDoS：目标是让服务不可用，不是病毒本身。
- SQL 注入：把恶意 SQL 拼进输入，骗后端数据库执行。

常见攻击类型的考试区分：

- 拒绝服务 DoS/DDoS：首要目标是让系统不能继续提供服务，例如耗尽带宽、连接数或服务器资源。
- 跨站脚本 XSS：把恶意脚本注入网页，让其他用户浏览页面时执行攻击者脚本，重点不是让服务停止。
- 信息篡改：攻击者修改传输或存储的信息，重点是完整性被破坏。
- 口令猜测：通过枚举、字典、撞库等方式猜用户密码，重点是获得未授权访问。
- 重放、IP 地址欺骗、拒绝服务都属于主动攻击，因为攻击者会伪造、修改、重发或干扰通信过程。
- 流量分析通常归为被动攻击，因为攻击者主要观察通信流量特征，不直接改动报文内容或阻断服务。

如果题干说“以被攻击对象不能继续提供服务为首要目标”，要选拒绝服务，而不是 XSS。XSS 是 Web 攻击，但它的典型目标是脚本执行、会话窃取、钓鱼或页面篡改。

### 7.2 代表真题：病毒与 DDoS

来源：`2022上半年选择题.md` 第8题

题干：下列不属于计算机病毒的是（D）。

- A. 永恒之蓝
- B. 蠕虫
- C. 特洛伊木马
- D. DDOS

答案：D

解析：DDoS 是分布式拒绝服务攻击，目标是耗尽服务资源，使系统不能继续提供服务。它是攻击方式，不是病毒或木马。题目把木马归入恶意代码类处理，考试中按题目语境判断即可。

## 8. 网络诊断与故障定位（权值：1/19）

常用命令题是低频但高性价比：

- ping：测试网络连通性，基于 ICMP 回显请求/应答。
- nslookup：查询 DNS 解析。
- netstat：查看网络连接、监听端口、路由等状态。
- telnet：远程登录，也可粗略测试某端口是否可连。
- ipconfig：查看本机 TCP/IP 配置，例如 IP 地址、网关、DNS 等；有些系统还可用它释放或更新 DHCP 地址。
- tracert/traceroute：跟踪到目标主机经过的路由节点，用于定位路径中哪一跳可能异常。

命令题的快速判断：

- “测连通性”选 ping。
- “查域名解析”选 nslookup。
- “看开放端口对应的连接或应用信息”选 netstat。
- “看本机 IP/网关/DNS 配置”选 ipconfig。
- “看路径经过哪些节点”选 tracert 或 traceroute。

故障题的基本判断：

- 全部网络软件都不通：看网卡、链路、IP 地址配置。
- 聊天软件能用但网页打不开：优先怀疑 DNS。
- 能解析域名但网页慢或打不开：再看 HTTP/HTTPS、代理、防火墙等。

## 9. 本章做题总口诀

1. 看到“在哪一层”，先问处理对象：信号、帧、IP 包、端口、应用数据。
2. 看到“端口号”，先把服务名写出来：HTTP 80、HTTPS 443、DNS 53、POP3 110、Telnet 23。
3. 看到“安全目标”，先选四目标：保密、完整、认证、不可否认。
4. 看到“证书”，先想身份信任；看到“摘要”，先想防篡改；看到“签名”，先想不可否认。
5. 看到“防火墙”，不要默认万能，先判断包过滤、应用级、Web 应用防火墙、IDS/IPS 的边界。
6. 看到“病毒/攻击”，先分恶意代码类型和攻击方式。
