## 2. 标准化基础与软件工程标准化（权值：1/51，约2%）

### 2.1 概念讲解

标准是对重复出现的事物、概念、过程或接口做出的统一规定。标准化不是一本文件本身，而是通过制定、发布、实施和复审标准，让重复性工作形成统一秩序的过程。

考试中要把三个词分清：

- 标准：已经形成的统一规定，例如国家标准、行业标准、企业标准。
- 规范或规程：标准的具体表现形式，常规定某类工作“应该怎么做”。
- 标准化：建立并执行这些规定的过程，目的通常是提高沟通效率、提高质量、降低协作成本，而不是直接提升某个开发人员个人技能。

常见分类可以从适用范围理解：

- 国际标准：例如 ISO、IEC。
- 国家标准：我国常见代号有 `GB`、`GB/T`、`GB/Z`。
- 行业标准：由行业主管部门或行业组织制定，例如电子、通信、军用等领域标准。
- 地方标准：在地方行政区域内适用。
- 企业标准：企业内部制定，用于约束企业自己的产品、流程或服务。
- 项目规范：项目内部为统一接口、文档、代码、测试、交付而制定的约束。

软件质量标准也会被放进标准化题里考。ISO/IEC 25010 属于软件产品质量模型相关标准，常见特性包括功能适合性、性能效率、兼容性、易用性、可靠性、安全性、可维护性、可移植性。可维护性关注软件被理解、分析、修改、测试和复用的难易程度，常见子特性有模块化、可重用性、可分析性、可修改性、可测试性；容错性属于可靠性方向，不属于可维护性。

### 2.2 解题流程

1. 先看题目问“标准化的作用”还是问“个人能力”。
2. 看到“沟通效率、管理水平、产品质量、流程一致性”，通常是标准化的正向作用。
3. 看到“提高每一位开发人员的开发技能”，要警惕。标准化能给团队提供统一规则，但不能保证每个人个人技能自动提升。
4. 若考标准编号，先识别 `GB/GB/T/GB/Z`、`ISO/IEC`、行业代号、地方 `DB`、企业 `Q`。
5. 若题目考 ISO/IEC 25010，可先判断选项属于哪类质量特性：可维护性看“模块化、分析、修改、测试、复用”，可靠性看“成熟性、可用性、容错、易恢复”。

### 2.3 常见陷阱

- 把“团队过程改进”误判成“个人技能提升”。
- 把推荐性国家标准 `GB/T` 当成强制性国家标准 `GB`。
- 把“软件工程标准化”理解成只写文档。它实际覆盖需求、设计、编码、测试、配置管理、质量保证、项目管理等多个活动。
- 把质量模型里的子特性张冠李戴。例如“容错性”听起来像系统好维护，但它实际描述系统出错后继续提供服务的能力，更接近可靠性，不是可维护性。

### 2.4 代表真题

来源：`2023下半年选择题.md` 第31题

题干：  
【考生回忆版】以下关于软件工程标准化的叙述中，不正确的是（D）。

- A. 可以提高开发人员之间的沟通效率
- B. 有助于提高管理水平
- C. 有助于提高软件产品质量
- D. 可以提高每一位开发人员的开发技能

答案：D

解析：  
软件工程标准化的对象是团队过程、协作规范、质量活动和管理流程。A、B、C 都属于标准化的典型收益。D 把“组织过程规范”偷换成“每个人个人技能一定提高”，这个因果过强，所以不正确。

## 3. 软件著作权归属：职务作品、委托开发、软件文档归属（权值：13/51，约25%）

### 3.1 概念讲解

这是第11章最高频考点。判断归属时，不要先看“谁写代码”，而要先看“是在什么法律关系下写的”。

职务软件或职务作品一般满足以下特征：

- 开发者在单位任职、实习或承担单位安排的开发任务。
- 使用了单位的工作条件、实验材料、技术资料、开发环境或不对外公开资料。
- 软件或软件文档是完成本职工作、项目任务或单位安排的成果。

在这类情形中，考试通常按“单位享有软件著作权或软件文档著作权”处理。开发者个人仍可能享有署名权等人身性权利，但不能据此带走源程序、拒不交还文档，或主张全部财产权归自己。

委托开发要抓住一句话：有约定按约定；无书面合同或约定不明，一般归受托开发方。题干中“甲公司受乙企业委托”“合同未明确约定归属”，通常答案是受托方甲公司，而不是出钱的委托方乙企业。

署名权要单独拎出来看。软件整体著作权归公司，不等于参与开发者的署名权可以被项目负责人随意删除；参与开发者离职、任务交接给别人，也不等于其对已完成贡献的署名资格消失。题目如果问“把王某从开发人员名单中删除并改成李某”，重点不是公司是否享有软件财产权，而是王某作为实际参与开发者的署名权是否被侵犯。

### 3.2 解题流程

1. 找身份：是任职人员、实习人员、项目组成员，还是外部受托方。
2. 找任务来源：是否由单位安排，是否属于本职工作。
3. 找资源来源：是否使用单位实验室、资料、设备或不公开技术资料。
4. 找合同约定：委托开发是否有书面合同，是否明确约定著作权归属。
5. 最后判断：单位/受托方享有著作权，还是个人享有署名权等人身权。

### 3.3 常见陷阱

- “他独立完成”不等于“著作权一定归他”。如果是在单位任务中完成，仍可能是职务成果。
- “已经辞职”不改变其在项目期间形成的署名权，也不改变单位对职务软件的财产权归属。
- “委托方出钱”不等于“无约定时归委托方”。无约定时通常归受托开发方。

### 3.4 代表真题一：职务软件文档

来源：`2020下半年选择题.md` 第12题

题干：  
小王是某高校的非全日制在读研究生，目前在甲公司实习，负责了该公司某软件项目的开发工作并撰写相关的软件文档。以下叙述中，正确的是（B）。

- A. 该软件文档属于职务作品，但小王享有该软件著作权的全部权利
- B. 该软件文档属于职务作品，甲公司享有该软件著作权的全部权利
- C. 该软件文档不属于职务作品，小王享有该软件著作权的全部权利
- D. 该软件文档不属于职务作品，甲公司和小王共同享有该著作权的全部权利

答案：B

解析：  
关键词是“在甲公司实习”“负责该公司软件项目”“撰写相关软件文档”。这是执行单位项目任务形成的软件文档，按职务作品处理。题目不是问小王是否参与创作，而是问软件文档著作权归属，所以选甲公司。

### 3.5 代表真题二：委托开发无约定

来源：`2022下半年选择题.md` 第11题

题干：  
以下关于某委托开发软件的著作权归属的叙述中，正确的是（C）。

- A. 该软件的著作权归属仅依据委托人与受托人在书面合同中的约定来确定
- B. 无论是否有合同约定，该软件的著作权都由委托人和受托人共同享有
- C. 若无书面合同或合同中未明确约定，则该软件的著作权由受托人享有
- D. 若无书面合同或合同中未明确约定，则该软件的著作权由委托人享有

答案：C

解析：  
委托开发不是“谁出钱就一定归谁”。先看合同，合同明确约定就按约定；没有书面合同或约定不明，考试口径通常是归受托人，即实际承接开发的一方。

## 4. 专利权、商标权、商业秘密权与其他权利边界（权值：12/51，约24%）

### 4.1 概念讲解

知识产权不是一种单一权利。软件相关题常把四类权利混在一起：

- 软件著作权：保护软件的表达，例如程序、文档、界面表达等；通常在开发完成时自动产生。
- 专利权：保护符合条件的发明创造，需要申请并获得授权；考试中常考“先申请原则”。
- 商标权：保护用于区分商品或服务来源的标志，需要注册；有效期满可以续展。
- 商业秘密权：保护不为公众知悉、具有商业价值、并采取保密措施的信息；只要秘密性和保密措施持续存在，就可能持续受保护。

商标权还有一个容易忽视的小考点：我国通常不强制所有商品都注册商标，但烟草类商品是考试中反复出现的例外。看到“必须使用注册商标”的选项，优先检查有没有“卷烟、雪茄烟、有包装的烟丝、烟草制品”这类关键词；医疗仪器、涂料、普通食品不能按这个例外处理。

专利权有地域性。某项技术只在美国取得专利，并不等于它自动在中国也受专利权保护；如果没有在中国申请并获得相应专利权，在中国制造、使用、销售通常不能仅凭美国专利来禁止。考试里看到“美国专利、未在中国申请、在中国销售”，要优先判断“不因中国销售向美国专利权人支付该专利许可费”；如果题干变成“返销美国”，才要重新考虑美国专利的地域效力。

### 4.2 解题流程

1. 题干出现“开发完成”，优先考虑软件著作权。
2. 题干出现“提出专利申请”，优先考虑专利申请权和先申请原则。
3. 题干出现“注册商标”“同日申请”“先使用”，优先考虑商标权规则。
4. 题干出现“保密约束、不公开资料”，优先考虑商业秘密。
5. 题干只说“注册商标”，没有说“申请专利”，不能自动推出享有专利权。
6. 题干出现“外国专利、未在中国申请”，先按专利地域性判断，别把外国专利当然扩张到中国。

### 4.3 常见陷阱

- 软件产品有商标，不等于软件技术有专利。
- 有保密措施，可以支持商业秘密权；但保密不等于已经获得专利权。
- 商标同日申请的前提是“同一种或类似商品上，相同或近似商标”。这里不要求两个商标字面完全相同；若读音、字形、含义或整体表现容易让公众混淆，也可能按近似商标处理。专利同日申请时通常协商确定。
- 商标注册还有限制性标志。县级以上行政区划地名通常不得作为商标注册，但地名如果还有其他含义，可能例外；题目常用“黄山”这类既是地名又有其他含义的词来设陷阱。
- 买到软件光盘，通常只取得光盘载体所有权或软件使用许可，不取得软件著作权或商标专用权。
- 专利同日申请和商标同日申请规则不同。专利同日申请通常协商确定申请人，不能因为一方先完成或先使用就直接归一方；同样的发明创造也不能同时授予两项专利。

### 4.4 代表真题一：软件、商标、商业秘密和专利边界

来源：`2022下半年选择题.md` 第13题

题干：  
M公司将其开发的某软件产品注册商标为S，为确保公司在市场竞争中占据优势地位，M公司对员工进行了保密约束，此情形下，该公司不享有（B）。

- A. 软件著作权
- B. 专利权
- C. 商业秘密权
- D. 商标权

答案：B

解析：  
软件开发完成后，M 公司可享有软件著作权；题干明确“注册商标为 S”，说明有商标权；题干又说对员工做了保密约束，支持商业秘密保护。但题干没有说申请并获得专利授权，所以不能推出专利权。

### 4.5 代表真题二：商标同日申请

来源：`2021上半年选择题.md` 第13题

题干：  
甲、乙两互联网公司于2020年7月7日就各自开发的库存管理软件分别申请“宏达”和“鸿达”商标注册，两个库存管理软件相似，甲第一次使用时间为2019年7月，乙第一次使用时间为2019年5月，此情形下，（D）能获准注册。

- A. “宏达”
- B. “宏达”和“鸿达”均
- C. 由甲、乙协商哪个
- D. “鸿达”

答案：D

解析：  
这题的关键不是“宏达”和“鸿达”字面完全相同，而是它们用于相似的库存管理软件，且读音相同、字形接近，考试按近似商标处理。两个申请人在同一种或类似商品上，以相同或近似商标同一天申请时，题目给了“第一次使用时间”，就要比较谁先使用。乙在 2019 年 5 月使用，早于甲的 2019 年 7 月，所以乙的“鸿达”更可能获准注册。

## 5. 软件著作权的产生、法律基础、具体权利（权值：9/51，约18%）

### 5.1 概念讲解

软件著作权保护的是软件作品。考试中要掌握三个层次：

- 产生时间：软件开发完成之日起自动产生，不以登记证书或首次发表为生效条件。
- 法律基础：常见考试口径是《中华人民共和国著作权法》和《计算机软件保护条例》。
- 权利内容：发表权、署名权、修改权、复制权、发行权、出租权、信息网络传播权、翻译权等。

我国保护计算机软件著作权的两个基本法律文件，考试通常认《中华人民共和国著作权法》和《计算机软件保护条例》。不要被《软件法》《版权法》这类选项带偏：我国没有作为考试答案的“软件法”；“版权法”是口语或泛称，不是本题口径下的基本法律文件名称。

软件著作权保护的是表达，不保护思想本身。具体到软件题，源程序、目标程序和软件文档属于保护对象；开发软件所用的思想、处理过程、操作方法、算法思想或数学概念本身，不属于软件著作权保护范围。看到“操作方法”“处理过程”“数学概念”，要警惕它们不是受软件著作权保护的对象。

国际上保护计算机软件知识产权的主要方式通常归入版权法或著作权法体系。这里的“版权法”是一个国际通用口径，强调通过版权/著作权机制保护软件表达；但在我国具体法律文件题里，仍应写《中华人民共和国著作权法》和《计算机软件保护条例》。两种问法不要混用。

软件著作权可以许可、转让，也可以在自然人死亡后由继承人继承其中的财产权利；但不是“所有权利”都能继承。署名权、开发者身份权等人身性权利通常不能像财产权一样由继承人继承。题目出现“自然人死亡后，继承人能继承所有权利”，要警惕“所有”这个绝对词。

署名权属于人身性很强的权利。即使软件整体财产权属于单位，参与开发者对其贡献仍可能享有署名权。软件翻译权容易出现教材口径和法规原文口径差异：如果题目明确问《计算机软件保护条例》第八条中的翻译权，应按法规原文理解为自然语言文字之间转换；如果题目按教材概念泛问软件翻译，才可能指程序设计语言转换。

### 5.2 解题流程

1. 问“何时产生”，选“开发完成之日”，不要选登记、计划文档、首次发表。
2. 问“保护软件著作权的基本法律文件”，找《著作权法》和《计算机软件保护条例》。
3. 问“开发者署名被删除”，优先判断侵犯署名权或开发者身份权。
4. 问“保护对象包括/不包括”，找源程序、目标程序、软件文档；排除思想、过程、操作方法和数学概念。
5. 问“翻译权”，先看题干有没有点名《计算机软件保护条例》。

### 5.3 常见陷阱

- 登记证书通常是证明材料，不是软件著作权产生条件。
- 首次公开发表不是软件著作权产生条件。
- 参与开发者离职，不代表其署名权消灭。
- 题干点名法规条文时，不要用模糊教材记忆覆盖法规原文。
- 购买正版软件不等于取得复制权。用户通常取得安装和使用许可，不能因此随意复制、发布或更改软件。
- “软件著作权属于软件开发者”这句话不能无条件成立。职务开发、委托开发、合作开发都有特殊归属规则；看到这种绝对表述，要回到题干关系判断。

### 5.4 代表真题一：产生时间

来源：`2025下半年选择题.md` 第5题

题干：  
【考生回忆版】以下关于软件著作权产生时间的叙述中，正确的是（D）。

- A. 自开发者获得软件著作权登记证书起
- B. 自开发计划文档完成时
- C. 自软件首次公开发表时
- D. 自软件开发完成之日起

答案：D

解析：  
软件著作权不是登记后才有，也不是发表后才有。开发完成是关键时点。登记证书主要用于证明权利，不能把“证明”误当成“产生”。

### 5.5 代表真题二：软件翻译权

来源：`2022下半年选择题.md` 第12题

题干：  
《计算机软件保护条例》第八条第一款第八项规定的软件著作权中的翻译权是将原软件由（D）的权利。

- A. 源程序语言转换成目标程序语言
- B. 一种程序设计语言转换成另一种程序设计语言
- C. 一种汇编语言转换成一种自然语言
- D. 一种自然语言文字转换成另一种自然语言文字

答案：D

解析：  
这题的关键不是“我平时怎么理解软件翻译”，而是题干明确限定了《计算机软件保护条例》第八条。按法规原文，翻译权是自然语言文字之间转换，所以选 D。A、B 更像编译或程序语言转换，不是本题指定条文下的答案。

## 6. 知识产权保护期限（权值：8/51，约16%）

### 6.1 概念讲解

保护期限题常用“永久、有限、可延长、无法定期限”设陷阱。

考试常用判断表：

| 权利 | 期限口径 |
|---|---|
| 著作权中的署名权、修改权、保护作品完整权 | 通常不受时间限制 |
| 发表权、复制权、发行权等财产权 | 有保护期 |
| 专利权 | 有明确法定期限，不是无限期 |
| 商标权 | 有有效期，但可以续展，因此可长期延续 |
| 商业秘密权 | 没有固定法定期限，取决于秘密性、价值性和保密措施是否持续 |

### 6.2 解题流程

1. 看到“永久保护”，优先找署名权、修改权、保护作品完整权。
2. 看到“可以延长”，优先找商标权续展。
3. 看到“具有法定时间性”，通常排除商业秘密权。
4. 看到“无限期拥有”，要区分：商标可续展，商业秘密无固定期限，但题目选项和问法不同，不能机械套一个答案。

### 6.3 常见陷阱

- 把商业秘密权说成“可以延长”。商业秘密不是到期续展，而是只要秘密状态和保密措施持续，就可能持续受保护。
- 把商标权说成“永久”。商标权本身有期限，但可以续展。
- 把著作权全部都说成永久。只有部分人身权通常不受时间限制，财产权不是永久。

### 6.4 代表真题一：法定时间性

来源：`2025上半年选择题.md` 第14题

题干：  
【考生回忆版】下列各项中，具有法定时间性的知识产权是（D）。  
①商业秘密权 ②专利权 ③商标权 ④著作权

- A. 仅①③
- B. 仅②④
- C. ①②③④
- D. 仅②③④

答案：D

解析：  
专利权、商标权、著作权都有法定期限或法定保护期规则。商业秘密权的关键不是固定期限，而是秘密性、价值性和保密措施是否继续存在，所以本题排除①。

### 6.5 代表真题二：著作权中不受限制的权利

来源：`2023上半年选择题.md` 第11题

题干：  
著作权中，（C）的保护期不受限制。

- A. 发表权
- B. 发行权
- C. 修改权
- D. 展览权

答案：C

解析：  
修改权属于著作人身权中通常不受时间限制的一类。发表权、发行权、展览权都不能简单视为永久。遇到“署名、修改、保护作品完整”要优先联想到不受时间限制。

## 7. 侵权、许可、合理使用、开源软件（权值：8/51，约16%）

### 7.1 概念讲解

这一类题的核心是判断“有没有权利人的许可”和“有没有法律允许的例外”。

常见规则：

- 未经许可复制、修改、发行、出租、通过网络传播他人软件，通常构成侵权。
- 技术论坛交流、稍加修改、删除原署名，不会自动变成合法使用。
- 购买侵权复制品且起初不知情，知道后仍要停止使用或销毁；若停止使用会造成重大损失，支付合理费用后可继续使用，但不能说“完全不侵权”。
- 软件许可分普通许可、独家许可、独占许可。独占许可下，权利人自己也不能使用；独家许可下，权利人自己仍可使用，但不能再许可第三人；普通许可下，权利人可再许可第三人且自己可使用。
- 合理使用不需要征得作者同意、不需要支付报酬，但通常限于已发表作品、适当引用、介绍评论等范围，不能构成自己作品的主要部分。
- 开源软件不是“没有著作权”。开源通常意味着著作权人通过开源许可允许他人按条件使用、修改、分发。
- 软件界面、代码、文档、图标和页面排布等具体表达，可能落入著作权保护；产品名称或标志才主要走商标权；技术方案是否受保护要看有没有专利授权。题干说“界面设计和功能基本一致”时，考试通常更偏向软件著作权侵权，而不是商标权或专利权。
- 论文或资料引用的合理使用，通常只能引用已经发表的作品，并且限于介绍、评论或说明问题的适当范围。引用未发表作品，或者引用内容构成自己作品的主要部分，就不能简单套“合理使用”。

### 7.2 解题流程

1. 先问是否有权利人许可。
2. 再问是否属于合理使用或软件合法持有人权利。
3. 如果题干有“修改后署自己名”“复制销售”“界面设计和功能基本一致”，优先按侵权判断。
4. 如果题干说“开源”，不要选“不涉及著作权”。
5. 如果题干问许可类型，抓两个条件：能否再许可第三人，权利人自己能否使用。

### 7.3 常见陷阱

- “放弃程序”不等于“著作权消灭”。
- “用于交流学习”不等于“可以随便修改发布”。
- “开源”不等于“归使用者所有”。
- “购买光盘”不等于“取得软件著作权”。
- “软件功能相似”要结合表达判断。功能思想本身不一定受著作权保护，但界面设计、代码、文档、图形等具体表达被高度复制时，考试会按著作权侵权方向判断。

### 7.4 代表真题一：修改并署名发布

来源：`2023下半年选择题.md` 第33题

题干：  
【考生回忆版】程序员甲将其编写完成的软件程序发给同事乙并进行讨论，之后由于甲对该程序极不满意，因此甲决定放弃该程序，后来乙将该程序稍加修改并署自己名在某技术论坛发布。下列说法中，正确的是（D）。

- A. 乙对该程序进行了修改，因此乙享有该程序的软件著作权
- B. 乙的行为没有侵犯甲的软件著作权，因为甲已放弃程序
- C. 乙的行为未侵权，因其发布的场合是以交流学习为目的的技术论坛
- D. 乙的行为侵犯了甲对该程序享有的软件著作权

答案：D

解析：  
甲把程序发给乙讨论，不等于许可乙修改后署名发布。甲对程序不满意或“放弃”，也不等于乙取得著作权。乙的行为同时踩到修改、署名、发布等风险点，按侵权判断。

### 7.5 代表真题二：开源软件

来源：`2025上半年选择题.md` 第17题

题干：  
【考生回忆版】关于开源软件的著作权，以下说法正确的是（D）。

- A. 开源软件不涉及著作权问题
- B. 开源软件的著作权归使用者所有
- C. 开源软件的著作权归开发者所有，使用者不能进行任何修改
- D. 开源软件的著作权归开发者所有，但使用者可以自由使用、修改和分发

答案：D

解析：  
开源的前提仍然是有人拥有著作权。开源许可把一部分使用、修改、分发权限开放出来，但通常要求遵守许可证条件。A 错在否认著作权，B 错在把权利归属转给使用者，C 错在忽略开源许可允许修改的特征。

## 8. 本章速记表

| 题干关键词 | 优先判断 |
|---|---|
| 开发完成之日起 | 软件著作权自动产生 |
| 登记证书 | 证明权利，不是产生条件 |
| 软件著作权基本法律文件 | 《中华人民共和国著作权法》和《计算机软件保护条例》 |
| 国际保护软件知识产权主要方式 | 版权法/著作权法体系 |
| 软件著作权保护对象 | 源程序、目标程序、软件文档；不保护思想、处理过程、操作方法、数学概念 |
| 软件著作权继承 | 继承人通常继承财产权利，不是继承署名权等所有权利 |
| 项目组、任职、实习、单位安排、使用单位资源 | 职务软件或职务作品，通常单位享有财产权 |
| 委托开发、合同未约定 | 通常受托方享有著作权 |
| 删除开发者姓名 | 侵犯署名权或开发者身份权 |
| 专利申请、先后申请 | 先申请者优先；同日申请通常协商 |
| 外国专利未在中国申请 | 专利有地域性，不能当然在中国主张该外国专利权 |
| 商标同日申请 | 前提是同一种或类似商品上的相同或近似商标；先使用者优先；都未使用时可协商或抽签 |
| 必须使用注册商标 | 重点记烟草制品，不要扩大到所有商品 |
| 注册商标有效期满 | 可续展 |
| 商业秘密 | 无固定法定期限，靠秘密性和保密措施维持 |
| 独家许可 | 不再许可第三人，但权利人自己可以用 |
| 独占许可 | 不再许可第三人，权利人自己也不能用 |
| 合理引用 | 通常限已发表作品、适当引用、非主要部分 |
| 开源软件 | 仍有著作权，只是按许可证开放使用、修改、分发 |
| 界面设计和表达被复制 | 优先考虑软件著作权侵权，不要只盯商标名称 |
| ISO/IEC 25010 可维护性 | 模块化、可重用性、可分析性、可修改性、可测试性；容错性属于可靠性 |

## 9. 本章学习建议

第11章不适合死背长段法律文字。更稳的方法是按题型建立判断链：

1. 先判断对象：软件、文档、发明、商标、商业秘密、论文引用。
2. 再判断关系：职务、委托、购买、许可、侵权复制、开源许可。
3. 再判断时点：开发完成、申请日、使用日、登记日、发表日。
4. 最后判断权利：著作权、署名权、专利申请权、商标权、商业秘密权、所有权或使用权。

本章第一轮训练应重点练“归属判断、期限判断、侵权和许可判断”；标准化由于本地真题稀少，本轮训练会在覆盖汇报中单独标出缺口。
