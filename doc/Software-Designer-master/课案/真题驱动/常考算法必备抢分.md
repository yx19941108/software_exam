# 常考算法必备抢分

## 生成口径

本课案只面向软件设计师下午 C 语言算法案例题的抢分。第一目标不是把算法学成竞赛水平，而是能在案例题第二问里快速写出：用了什么算法思想、为什么、时间复杂度是多少。

本课案统计口径：

- 完整本地案例题样本：`2015上半年案例题.md` 到 `2023上半年案例题.md` 中命中的 C 语言算法案例题。
- 2024 下半年：本地案例题只有 2 道，未命中算法案例题。
- 2025 上半年：本地题源为考生回忆版，命中“回溯法，子序列之和”，但无标准答案和解析，只作为补充观察，不计入完整样本权重。
- 教材补充口径：持久化教材目录确认第 8 章包含 `分治法、动态规划法、贪心法、回溯法、分支限界法、概率算法、近似算法、数据挖掘算法、智能优化算法`。案例题没有覆盖的项，按备考常见问法补齐。

### 2. 覆盖结论

完整案例题中最高频的是动态规划，尤其是二维表和区间 DP。能不能拿分，主要看三件事：

- 能不能从题干里的“最优子结构、递归式、保存中间结果”判断为动态规划。
- 能不能按循环层数或递归规模判断复杂度。
- 能不能把 `c[i][j]`、`m[i][j]`、`cost[i][j]` 这类表的含义翻译成白话。

### 3. 一句话背诵表

| 算法 | 一句话思想 | 复杂度高频口径 | 识别关键词 |
| --- | --- | --- | --- |
| 动态规划 | 大问题最优解由子问题最优解组合，用表保存重复子问题 | 二维表常 `O(mn)`；区间 DP 常 `O(n^3)`；0-1 背包常 `O(nT)` | 最优子结构、递归式、数组保存中间结果 |
| 回溯 | 在解空间树中尝试选择，不满足约束就撤销并换下一种 | n 皇后/子集常指数级；哈密顿回路常 `O(n!)` | 尝试、冲突、退回、深度优先 |
| 分治 | 把问题拆成同类小问题，递归求解后合并 | 二分型常 `O(log n)`；归并/快排平均 `O(nlogn)` | 划分、递归、规模减半、合并 |
| 贪心 | 每一步选择当前最优，并且该选择能导向全局最优 | 排序后选择常 `O(nlogn)`；Dijkstra 常 `O(n^2)` 或堆优化 `O(ElogV)` | 当前最优、单位价值、最短边、最短距离 |
| 分支限界 | 在解空间树上搜索，用界函数剪掉不可能更优的分支 | 最坏仍指数级，剪枝后平均更快 | 队列、优先队列、上界、下界、活结点 |
| KMP | 主串不回退，模式串用 `next[]` 跳转 | `O(lt+ls)` | next 数组、主串、子串、失配 |
| 希尔排序 | 按增量分组做插入排序，增量逐步缩小到 1 | 题源口径约 `O(n^1.3)`，小于 `O(n^2)`；不稳定 | 增量、分组、直接插入 |
| 堆排序 | 建大顶堆，堆顶与末尾交换，再调整堆 | `O(nlogn)`，不稳定 | 大顶堆、Heapify、下滤 |
| DFS | 沿一条路径走到底，再回退访问其他邻接点 | 邻接表 `O(V+E)` | visited、邻接表、递归访问 |
| 二分查找 | 每次比较中间元素，把查找区间减半 | `O(log n)` | 有序表、中间位置、折半 |
| 哈夫曼 | 每次合并权值最小的两个结点，生成最优前缀编码 | 建树常 `O(nlogn)` | 权值、最优二叉树、前缀编码 |
| 分数背包 | 按单位重量价值从高到低拿，最后一个可拿一部分 | 排序主导 `O(nlogn)` | 物品可分割、单位价值 |
| 0-1 背包 | 每个物品只有选/不选，用容量表保存最优值 | `O(nT)` | 不可分割、选或不选、容量 |
| 概率算法 | 算法步骤引入随机选择，通常用概率换平均性能 | 随机快排平均 `O(nlogn)` | 随机、概率、期望 |
| 近似算法 | 对难以精确求最优的问题，求有保证的近似解 | 依具体问题，常多项式时间 | 近似比、NP 难、可接受解 |
| k-means | 随机选中心，按最近中心分组，再更新中心 | `O(t*k*n*d)` | 聚类、中心点、迭代 |
| 遗传算法 | 种群中反复选择、交叉、变异，逐代逼近较优解 | `O(代数*种群规模*适应度计算)` | 选择、交叉、变异、适应度 |

## 第二章：典型 C 代码模板

下面代码是背诵模板，不是唯一写法。考试时优先按题干变量名改写。

### 1. 动态规划：0-1 背包

```c
int knapsack(int w[], int v[], int n, int T) {
    int c[101][1001] = {0};
    int i, j;

    for (i = 1; i <= n; i++) {
        for (j = 0; j <= T; j++) {
            c[i][j] = c[i - 1][j];
            if (j >= w[i] && c[i][j] < c[i - 1][j - w[i]] + v[i]) {
                c[i][j] = c[i - 1][j - w[i]] + v[i];
            }
        }
    }
    return c[n][T];
}
```

背：二维表 `c[i][j]` 表示前 `i` 个物品、容量为 `j` 的最大价值。复杂度 `O(nT)`。

### 2. 动态规划：最长公共子串

```c
int longest_common_substring(char x[], int m, char y[], int n) {
    int c[101][101] = {0};
    int i, j, ans = 0;

    for (i = 1; i <= m; i++) {
        for (j = 1; j <= n; j++) {
            if (x[i - 1] == y[j - 1]) {
                c[i][j] = c[i - 1][j - 1] + 1;
                if (ans < c[i][j]) {
                    ans = c[i][j];
                }
            } else {
                c[i][j] = 0;
            }
        }
    }
    return ans;
}
```

背：相等才从左上角加 1，不相等归 0。复杂度 `O(mn)`。

### 3. 动态规划：编辑距离

```c
int min3(int a, int b, int c) {
    int m = a < b ? a : b;
    return m < c ? m : c;
}

int edit_distance(char a[], int m, char b[], int n) {
    int d[101][101];
    int i, j;

    for (i = 0; i <= m; i++) {
        d[i][0] = i;
    }
    for (j = 0; j <= n; j++) {
        d[0][j] = j;
    }
    for (i = 1; i <= m; i++) {
        for (j = 1; j <= n; j++) {
            if (a[i - 1] == b[j - 1]) {
                d[i][j] = d[i - 1][j - 1];
            } else {
                d[i][j] = min3(d[i - 1][j] + 1, d[i][j - 1] + 1, d[i - 1][j - 1] + 1);
            }
        }
    }
    return d[m][n];
}
```

背：删除、插入、替换三者取最小。复杂度 `O(mn)`。

### 4. 动态规划：矩阵链乘 / 区间 DP

```c
int matrix_chain(int p[], int n) {
    int cost[101][101] = {0};
    int len, i, j, k, temp;

    for (len = 2; len <= n; len++) {
        for (i = 1; i <= n - len + 1; i++) {
            j = i + len - 1;
            cost[i][j] = 0x3fffffff;
            for (k = i; k < j; k++) {
                temp = cost[i][k] + cost[k + 1][j] + p[i - 1] * p[k] * p[j];
                if (temp < cost[i][j]) {
                    cost[i][j] = temp;
                }
            }
        }
    }
    return cost[1][n];
}
```

背：区间长度 `len`、左端 `i`、右端 `j`、断点 `k`，三层循环，所以 `O(n^3)`。

### 5. 回溯：n 皇后

```c
int n;
int queen[20];
int count;

int place(int row) {
    int i;
    for (i = 1; i < row; i++) {
        if (queen[i] == queen[row] || abs(queen[i] - queen[row]) == row - i) {
            return 0;
        }
    }
    return 1;
}

void nqueen(int row) {
    int col;
    if (row > n) {
        count++;
        return;
    }
    for (col = 1; col <= n; col++) {
        queen[row] = col;
        if (place(row)) {
            nqueen(row + 1);
        }
    }
}
```

背：每行尝试一列，列冲突或斜线冲突就不继续。思想是回溯法。

### 6. 回溯：子序列之和

```c
int subset_sum(int a[], int n, int idx, int sum, int target) {
    if (sum == target) {
        return 1;
    }
    if (idx == n || sum > target) {
        return 0;
    }
    if (subset_sum(a, n, idx + 1, sum + a[idx], target)) {
        return 1;
    }
    return subset_sum(a, n, idx + 1, sum, target);
}
```

背：每个数只有选和不选两条路。最坏复杂度 `O(2^n)`。

### 7. 分治：假币问题

```c
int fake_coin(int coin[], int left, int right) {
    int mid, i;
    int left_sum = 0;
    int right_sum = 0;

    if (left == right) {
        return left;
    }
    mid = (left + right) / 2;
    for (i = left; i <= mid; i++) {
        left_sum += coin[i];
    }
    for (i = mid + 1; i <= right; i++) {
        right_sum += coin[i];
    }
    if (left_sum < right_sum) {
        return fake_coin(coin, left, mid);
    }
    return fake_coin(coin, mid + 1, right);
}
```

背：每次只保留较轻的一半，问题规模约减半，所以 `O(log n)`。奇数个硬币时要单独判断中间硬币。

### 8. KMP 匹配

```c
int kmp(char t[], char s[], int next[], int lt, int ls) {
    int i = 0;
    int j = 0;

    while (i < lt && j < ls) {
        if (j == -1 || t[i] == s[j]) {
            i++;
            j++;
        } else {
            j = next[j];
        }
    }
    if (j >= ls) {
        return i - ls;
    }
    return -1;
}
```

背：主串 `i` 不回退，子串 `j` 按 `next[j]` 回跳。复杂度 `O(lt+ls)`。

### 9. 希尔排序

```c
void shell_sort(int a[], int n) {
    int gap, i, j, temp;

    for (gap = n / 2; gap > 0; gap = gap / 2) {
        for (i = gap; i < n; i++) {
            temp = a[i];
            for (j = i - gap; j >= 0 && temp < a[j]; j = j - gap) {
                a[j + gap] = a[j];
            }
            a[j + gap] = temp;
        }
    }
}
```

背：按增量分组做插入排序，不稳定。题源口径：复杂度小于 `O(n^2)`，约 `O(n^1.3)`。

### 10. 堆排序核心：下滤

```c
void heapify(int r[], int v, int n) {
    int i = v;
    int j = 2 * i;
    r[0] = r[i];

    while (j <= n) {
        if (j < n && r[j] < r[j + 1]) {
            j++;
        }
        if (r[0] < r[j]) {
            r[i] = r[j];
            i = j;
            j = 2 * i;
        } else {
            break;
        }
    }
    r[i] = r[0];
}
```

背：`r[0]` 保存原父结点，较大的孩子一路上提，最后 `r[i] = r[0]`。堆排序复杂度 `O(nlogn)`。

### 11. DFS 连通分量

```c
void dfs(int v, int graph[][100], int visited[], int n) {
    int i;
    visited[v] = 1;
    for (i = 0; i < n; i++) {
        if (graph[v][i] && !visited[i]) {
            dfs(i, graph, visited, n);
        }
    }
}

int component_count(int graph[][100], int n) {
    int visited[100] = {0};
    int i, count = 0;
    for (i = 0; i < n; i++) {
        if (!visited[i]) {
            count++;
            dfs(i, graph, visited, n);
        }
    }
    return count;
}
```

背：外层每发现一个未访问顶点，连通分量数加 1，再 DFS 标记整块。邻接表复杂度 `O(V+E)`，邻接矩阵 `O(V^2)`。

### 12. 贪心：分数背包

```c
typedef struct {
    double w;
    double v;
    double rate;
} Item;

double fractional_knapsack(Item a[], int n, double cap) {
    int i;
    double ans = 0;

    /* 考试若不要求排序代码，可直接写“按 rate 从大到小排序”。 */
    sort_by_rate_desc(a, n);

    for (i = 0; i < n && cap > 0; i++) {
        if (a[i].w <= cap) {
            ans += a[i].v;
            cap -= a[i].w;
        } else {
            ans += a[i].rate * cap;
            cap = 0;
        }
    }
    return ans;
}
```

背：分数背包可切割，所以按单位价值贪心能最优；排序主导复杂度 `O(nlogn)`。

### 13. 分支限界：0-1 背包框架

```c
typedef struct {
    int level;
    int weight;
    int value;
    double bound;
} Node;

void branch_bound_knapsack(void) {
    /* 1. 按单位价值排序物品。 */
    /* 2. 根结点入队，记录当前最好值 best。 */
    /* 3. 每次取出一个活结点，扩展“选当前物品”和“不选当前物品”。 */
    /* 4. 如果结点 bound <= best，剪枝；否则入队继续搜索。 */
}
```

背：分支限界不是复杂度变成多项式，最坏仍可能指数级；它靠上界/下界剪掉不可能更优的分支。

### 14. 概率算法：随机化快排

这里重点背“随机选基准”这一点，`swap` 和 `partition` 按普通快速排序辅助函数理解即可。

```c
void quick_sort(int a[], int left, int right) {
    int pivot_index;
    if (left >= right) {
        return;
    }
    pivot_index = left + rand() % (right - left + 1);
    swap(a, left, pivot_index);
    pivot_index = partition(a, left, right);
    quick_sort(a, left, pivot_index - 1);
    quick_sort(a, pivot_index + 1, right);
}
```

背：随机选基准降低遇到最坏划分的概率。平均 `O(nlogn)`，最坏仍 `O(n^2)`。

### 15. 近似算法：顶点覆盖 2-近似框架

```c
void vertex_cover_approx(void) {
    /* while 图中还有边 (u, v):
       选择这条边；
       把 u 和 v 加入覆盖集合；
       删除所有与 u 或 v 相连的边；
    */
}
```

背：近似算法用于精确最优代价太高的问题，追求有保证的可接受解。不要把“近似算法”说成“随便猜”。

### 16. 数据挖掘：k-means 框架

```c
void kmeans(void) {
    /* 1. 选择 k 个初始中心。
       2. 对每个样本，分配到最近中心。
       3. 对每一类，重新计算中心。
       4. 重复 2-3，直到中心不再明显变化或达到迭代次数。
    */
}
```

背：复杂度常按 `O(t*k*n*d)`，其中 `t` 是迭代次数，`k` 是簇数，`n` 是样本数，`d` 是特征维数。

### 17. 智能优化：遗传算法框架

```c
void genetic_algorithm(void) {
    /* 1. 初始化种群。
       2. 计算每个个体的适应度。
       3. 按适应度选择父代。
       4. 交叉产生子代。
       5. 随机变异。
       6. 形成新种群，循环直到达到代数或满意解。
    */
}
```

背：遗传算法的关键词是选择、交叉、变异、适应度。复杂度常按 `O(代数 * 种群规模 * 适应度计算代价)`。

## 最短背诵清单

如果时间不够，只背下面 8 条：

1. 动态规划：最优子结构 + 重叠子问题 + 表保存中间结果。
2. 二维 DP 复杂度常看两层循环：`O(mn)` 或 `O(nT)`。
3. 区间 DP 有 `len/i/k` 三层循环：常 `O(n^3)`。
4. 回溯：尝试、冲突、退回，最坏通常指数级。
5. 分治：划分同类子问题，规模减半常 `O(log n)`，排序常 `O(nlogn)`。
6. 贪心：每步当前最优；分数背包可贪心，0-1 背包不能简单贪心。
7. KMP：主串不回退，`next[]` 回跳，`O(lt+ls)`。
8. 堆排序：建大顶堆 + 下滤，`O(nlogn)`，不稳定。
