## 一、线性表、栈与队列（权值：5/20，约25.0%）

### 2.1 这个知识点到底在讲什么

这一块是数据结构最基础的一章，主要考：

- 线性表
- 单链表 / 循环链表
- 栈
- 队列

### 2.2 核心概念详细讲解

`线性表` 的核心特征：

- 除首尾外，每个元素只有一个直接前驱和一个直接后继

`单链表`：

- 结点至少包含数据域和指针域
- 逻辑相邻不一定物理相邻

`循环链表`：

- 尾结点指针不再是 `NULL`
- 而是回到头结点或首元结点

`栈`：

- 后进先出

`队列`：

- 先进先出

链队列题常考：

- `front` 和 `rear` 的含义
- 空队列边界

### 2.3 易错点

1. 把顺序表的随机访问能力套到单链表上。
2. 忘记循环链表尾指针会回连。
3. 把栈和队列的进出顺序记反。

### 2.4 真题示例

题目：

以下关于单链表存储结构特征的叙述中，错误的是（A）。

- A. 单链表支持随机访问
- B. 单链表结点间的逻辑关系通过指针来体现
- C. 单链表插入和删除操作通常不需要移动大量元素
- D. 单链表中逻辑相邻的结点物理地址不一定连续

正确答案：`A`

详细解析：

- 单链表最大的短板就是不支持随机访问。
- 因为它不是按下标直接定位，而是必须顺着指针一个个往后找。
- B 对：链表靠指针域体现结点关系。
- C 对：链表插删通常比顺序表少“搬动元素”的代价。
- D 对：链表逻辑相邻不等于物理相邻。

## 二、数组、矩阵与压缩存储（权值：5/20，约25.0%）

### 3.1 这个知识点到底在讲什么

矩阵题最常见的是：

- 对称矩阵
- 上三角 / 下三角矩阵
- 稀疏矩阵
- 压缩存储位置映射

### 3.2 核心概念详细讲解

压缩存储的本质是：

- 不再为“重复值”或“固定为 0 的位置”浪费空间

对称矩阵能压缩，因为：

- `a[i][j] = a[j][i]`

上三角矩阵能压缩，因为：

- 另一半通常固定为同一值（常见为 0）

稀疏矩阵能压缩，因为：

- 非零元素很少

### 3.3 真题示例 1

题目：

某 `n` 阶的三对角矩阵 `A` 如下图所示，按行将元素存储在一维数组 `M` 中，设 `a1,1` 存储在 `M[1]`，那么 `ai,j`（`1<=i,j<=n` 且 `ai,j` 位于三条对角线中）存储在 `M（D）`。

- A. `i+2j`
- B. `2i+j`
- C. `i+2j-2`
- D. `2i+j-2`

正确答案：`D`

详细解析：

- 这类压缩存储题最稳的解法不是背公式，而是代最简单的已知位置验证。
- 已知 `a1,1 -> M[1]`，代入四个选项排除不符合的。
- 再看下一项 `a1,2 -> M[2]`，继续验证。
- 只有 `D` 同时满足。

所以矩阵压缩存储的常规做法是：

1. 先拿题目给的基准位置代入。
2. 再用相邻元素位置继续验证。
3. 不要一上来死背公式。

### 3.4 真题示例 2

题目：

某文件系统采用多级索引结构。若磁盘块大小为 `1K` 字节，每个块号占 `3` 字节，那么采用二级索引时的文件最大长度为（C）K字节。

- A. 1024
- B. 2048
- C. 116281
- D. 232562

正确答案：`C`

详细解析：

- 一个索引块里能放多少地址，先算 `1024/3`。
- 二级索引表示“索引块里的地址再指向索引块”，所以规模要平方。
- 最后乘以每块能表示的数据块大小 `1KB`。

## 三、树与二叉树（权值：4/20，约20.0%）

### 4.1 这个知识点到底在讲什么

树题通常考：

- 树的度与叶子结点数
- 二叉树遍历
- 平衡二叉树
- 哈夫曼树

### 4.2 核心概念详细讲解

树中有一个很重要的守恒关系：

- 所有结点出度之和 = 边数 = 结点总数 - 1

所以给你不同度数结点数时，可以反推出叶子数。

`平衡二叉树`：

- 任一结点左右子树高度差不超过 `1`

`哈夫曼树`：

- 带权路径长度最小的二叉树
- 常用来构造前缀编码

### 4.3 真题示例

题目：

某树 `T` 的度为 `4`，其中有 `5` 个度为 `4` 的结点，`8` 个度为 `3` 的结点，`6` 个度为 `2` 的结点，`10` 个度为 `1` 的结点，则 `T` 中的叶子结点个数为（A）。

- A. 38
- B. 29
- C. 66
- D. 57

正确答案：`A`

详细解析：

- 先算出度总和：
  - `5×4 + 8×3 + 6×2 + 10×1 = 66`
- 树中边数 = 结点总数 - 1
- 非叶子结点总数：
  - `5 + 8 + 6 + 10 = 29`
- 设叶子结点为 `x`
- 则：
  - `66 = (29 + x) - 1`
  - `66 = 28 + x`
  - `x = 38`

## 四、查找与排序（权值：4/20，约20.0%）

### 5.1 这个知识点到底在讲什么

查找与排序题主要考：

- 折半查找
- 排序稳定性
- 哪种算法第一趟能确定元素位置
- 构堆结果

### 5.2 真题示例

题目：

对于一个初始无序的关键字序列，在下面的排序方法中，（C）第一趟排序结束后，一定能将序列中的某个元素在最终有序序列中的位置确定下来。

- A. ①②③⑥
- B. ①②③⑤⑥
- C. ②③④⑤
- D. ③④⑤⑥

正确答案：`C`

详细解析：

- 冒泡排序：第一趟后至少能确定一个最大元素位置。
- 简单选择排序：第一趟后能确定一个最小元素位置。
- 堆排序：每轮调整后能确定堆顶极值位置。
- 快速排序：每轮划分后基准元素位置确定。
- 归并排序第一趟后通常不能确定最终位置。

## 五、图（权值：2/20，约10.0%）

### 6.1 这个知识点到底在讲什么

图在第3章里不是权值最高，但也会稳定出现。

最常见是：

- 图的基本概念
- 邻接矩阵 / 邻接表
- 最小生成树
- 最短路径

### 6.2 真题示例

题目：

图 `G` 的邻接矩阵如下图所示（顶点依次表示为 `v0、v1、v2、v3、v4、v5`），`G` 是（B/A）。对 `G` 进行广度优先遍历（从 `v0` 开始），可能的遍历序列为（ ）。

问题1：

- A. 无向图
- B. 有向图
- C. 完全图
- D. 强连通图

问题2：

- A. `v0、v1、v2、v3、v4、v5`
- B. `v0、v2、v4、v5、v1、v3`
- C. `v0、v1、v3、v5、v2、v4`
- D. `v0、v2、v4、v3、v5、v1`

正确答案：`B、A`

详细解析：

- 先看邻接矩阵是否对称：
  - 若 `v0->v1` 存在，但 `v1->v0` 不存在，就说明它不是无向图，而是有向图。
- 再做广度优先遍历：
  - 从 `v0` 出发
  - 先访问它的邻接点
  - 再按“先访问的结点的邻接点先入队”的顺序继续扩展
- 因此问题1选 `B`，问题2选 `A`。

这类图题最容易错的地方是：

- 把“有路径”误看成“有边”
- 把深度优先和广度优先的访问顺序混掉
