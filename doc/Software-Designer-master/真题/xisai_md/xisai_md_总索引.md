# 软件设计师希赛真题 Markdown 总索引

## 用途定位

本索引是后续软件设计师课程出题时的题库指引知识库。

后续使用规则：

1. 出题前，优先通过本索引定位可用真题文件，再进入对应 Markdown 搜题。
2. 优先复用本索引已覆盖的 `2015-2025` 希赛真题 Markdown，不要先跳去网页临时找题。
3. 若本索引对应知识库中的题目确实无法满足当前出题要求，再咨询游工是否补充新题源或扩库。
4. 对于 `2023下半年`、`2024下半年案例题`、`2025上半年案例题` 这类站点原始内容偏回忆版或题量较少的卷，出题前要先看原文件内容完整度，不要只看年份想当然。

## 搜题建议

推荐检索顺序：

1. 先按年份与题型缩小范围。
2. 再打开对应文件，用 `知识点关键词` 搜题。
3. 选择题优先查 `基础知识` 卷，案例题优先查 `应用技术` 卷。
4. 如果目标是讲课练习题，优先选 `2019-2025` 的近年卷；如果目标是补薄弱点，可回补到 `2015-2018`。

常见检索关键词示例：

- 选择题：`操作系统`、`数据库`、`UML`、`设计模式`、`数据流图`、`哈夫曼`、`Cache`、`软件著作权`
- 案例题：`DFD`、`ER图`、`关系模式`、`UML`、`设计模式`、`算法`、`数据流图`
