- 目标不是复述聊天记录，而是让后续会话按本文档直接恢复“第一章拉题”能力。

  - 优先换年份补新题

  - 页面对象 `images=1`

### 当前可用做法

- 已验证目录关键页可读：

## 失败路径总结

### 1. 直接读本地教材 PDF 文本

- 失败原因：
  - 扫描版，无有效文本层
- 结论：
  - 不要继续把它当可直接抽文本的 PDF
  - 要么转图片人工读，要么只用目录页做章节定位

### 2. 直接读本地真题 PDF 文本

- 路径：
  - `doc/Software-Designer-master/真题/*.pdf`
- 现象：
  - `pdfplumber` 可抽到文本，但大量中文出现字体映射乱码
- 结论：
  - 可作为“题面存在性”来源
  - 不适合作为高质量聊天出题文本主来源
  - 如果必须使用，优先渲染成图片再人工读题

### 3. 猜 educity PDF 直链

- 失败原因：
  - 多数直链请求返回 `text/html`
  - 或在浏览器里落到 `404页面 - 您访问的页面读书去了`
- 即使浏览器 PDF 查看器能打开某些页面：
  - `requests` 抓到的仍可能是 HTML 包装页
  - `chrome-devtools` 网络请求保存到本地的也可能是 HTML wrapper
- 结论：
  - 不再以“猜年份直链 PDF”作为主路径

### 4. 浏览器 PDF 查看器下载

- 这是“能用但不稳”的次优路径。
- 特点：
  - 需要页面真的打开到 PDF viewer
  - 还需要点一次下载按钮，文件落在 `Downloads`
  - 下载后再用 `pdfplumber / pypdfium2` 二次处理
- 结论：
  - 仅当题库页面拿不到题时再用

## 3. 真题页抓题方法

### 页面特征

- 进入练习模式后，URL 类似：
  - `https://wangxiao.xisaiwang.com/tiku2/exam534880820.html`
- 页面正文中能直接看到：
  - `第1题（单选题）：`
  - `上一题`
  - `下一题`
  - `查看答案与解析`

### 已验证的 DOM 翻页点

- “下一题”所在容器可通过：
  - `div.bRightWrap`
  来点击

### 已验证的题干提取方式

- 可直接对 `document.body.innerText` 做正则提取：
  - `第(\\d+)题（单选题）：([\\s\\S]*?)标记`
- 再把换行压平后保留前 `200-300` 字，足够做筛题

### 已验证的批量抓题思路

- 在练习模式页循环：
  1. 读取当前题号和题干
  2. 记录到数组
  3. 点击“下一题”
  4. 等待题号变化
  5. 重复直到整卷结束

### 推荐的批量抓题脚本思路

```javascript
async () => {
  const parse = () => {
    const txt = document.body.innerText;
    const m = txt.match(/第(\d+)题（单选题）：([\s\S]*?)标记/);
    return m ? {
      num: Number(m[1]),
      text: m[2].trim().replace(/\n+/g, ' ').slice(0, 220),
    } : null;
  };

  const nextBtn = () =>
    [...document.querySelectorAll('div.bRightWrap')]
      .find(el => (el.innerText || '').includes('下一题'));

  const out = [];
  const seen = new Set();

  for (let i = 0; i < 80; i++) {
    const q = parse();
    if (q && !seen.has(q.num)) {
      out.push(q);
      seen.add(q.num);
    }

    const btn = nextBtn();
    if (!btn) break;

    const prevNum = q ? q.num : -1;
    btn.click();

    const start = Date.now();
    while (Date.now() - start < 4000) {
      await new Promise(r => setTimeout(r, 120));
      const nq = parse();
      if (nq && nq.num !== prevNum) break;
    }
  }

  return out;
}
```

## 去重规则

- 对同一知识点的不同问法可以保留
- 对“几乎同题面”的题必须去重
- 若新题量不足：
  - 允许极少量重复
  - 重复率不得超过 `20%`
- 若仍凑不满：
  - 直接报告实际找到数量
  - 不强行凑满

## 图示题处理规则

- 如果题干依赖图示而 DOM 文本无法完整表达：
  - 优先使用浏览器页内现成图示继续出题
  - 若无法快速稳定复原图示，则放弃该题
- 不要再出现“题目缺图但仍照常出给用户”的情况
