### 练习 3

- 判定：未达标
- 建议分：`2 / 5`
- 已答对：
  - 你意识到：
    - 第 `5` 条新需求确实在改动原有建模假设
  - 也意识到：
    - 这不是一个纯粹的界面层问题
- 失分点：
  - 你把新需求理解成：
    - “活动群加入上级群，本质上还是用户成员关系，只要给活动群类增加一个重写方法”
  - 这在下午题口径里失分很重
  - 原因是：
    - 题目考的是类图结构是否要支持“群体也能成为成员”
    - 不是只考某个方法的参数类型能不能重载
  - 也就是说，这题真正要你回答的是：
    - 原模型中“成员只能是用户”这条假设失效了
    - 因而应优先考虑修改成员关系的抽象层级，或引入更一般的成员概念
    - 至少也要改类之间的关系，而不是只改方法签名
- 更标准口径：
  - 这类题更稳的答法应是：
    - 原有类图不够，因为“成员”的取值范围从“用户”扩大到了“用户或群体”
    - 因此优先要改的是：
      - `关系`
      - 甚至是 `抽象层级`
    - 而不是只在某个类上补一个重载方法
- 结论：
  - 你抓到了“新需求会影响设计”
  - 但还没抓到这道题最关键的点：它影响的是模型结构，不只是接口实现
