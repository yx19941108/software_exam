## 逐题批改

### 练习 1

- 判定：基本通过
- 建议分：`5 / 6`
- 已答对：
  - 你正确识别了参与者：
    - `学生`
    - `教师`
    - `教务员`
  - 你列出的核心用例基本成立：
    - `查询课程`
    - `提交选课申请`
    - `查看选课名单`
    - `审核特殊选课申请`
  - 你把“发送提醒”判断为系统内部结果，这个方向是对的
- 失分点：
  - 你把 `登录` 也列为用例，这不算大错，但这题的稳定主线更应聚焦题干明确业务目标本身，而不是默认把所有系统通用入口动作都并列抬成核心用例
  - 你对“发送提醒”的理由里写了：
    - “即使没有此步骤系统也能够运行，不阻塞系统主业务流程”
  - 这句话不够稳
- 更标准口径：
  - 判断它不是独立参与者的关键，不在于“删掉后系统能不能运行”
  - 而在于：
    - 它不是系统外部角色
    - 它也不是由外部角色单独发起的独立业务目标
    - 更像某个主用例完成后的系统响应或附带行为

### 练习 2

- 判定：未达标
- 建议分：`3 / 6`
- 已答对：
  - 你知道这里至少应抽出：
    - `读者`
    - `图书`
    - `借阅单`
  - 也意识到 `借书 / 还书 / 续借` 不能简单当普通字段看待
- 失分点：
  - 你把类写成了：
    - `读者信息`
    - `图书信息`
    - `借阅单信息`
  - 这不够考试化
  - 更稳的类名应直接是：
    - `读者`
    - `图书`
    - `借阅单`
  - 你把：
    - `读者动作（借书、还书、续借）`
    写进了属性列表，这一步是明显失分点
  - 第 3 问里你又说：
    - 先设置一个“读者动作”枚举属性
    - 同时这三个对读者来说是用例，对类来说是操作
  - 这在考试表达里是混乱的，因为同一轮回答里你把它同时落成了：
    - 属性
    - 用例
    - 操作
- 更标准口径：
  - 本题最稳回答应是：
    - `借书 / 还书 / 续借` 更像业务动作
    - 在需求视角下更接近用例
    - 在类的职责表达里更接近操作
    - 不应优先写成普通属性
- 结论：
  - 你已经抓到“它不是普通静态字段”这一层
  - 但还没把 `属性 / 用例 / 操作` 三者的边界讲清楚

### 练习 3

- 判定：完全正确
- 建议分：`7 / 8`
- 已答对：
  - 你正确判断了：
    - `配送员` 与 `员工` 是泛化关系

- 结论：
  - 这题主线你已经掌握了
  - 扣分主要是术语方向的小精度问题
