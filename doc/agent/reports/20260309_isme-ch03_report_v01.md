# 第3章 程序设计语言

## 教材定位
- PDF 目录起始页：59
- 目录关键词：程序语言基础、控制结构、数据表示、过程与函数、面向对象

## 你要掌握的核心知识
- 掌握程序设计语言的基本概念、语法与语义、编译与解释的区别。
- 理解顺序、选择、循环、递归等控制结构以及数组、记录、指针等数据表示方式。
- 掌握过程、函数、参数传递、作用域、存储期等基本程序设计机制。
- 理解抽象、封装、继承、多态等面向对象思想。

## 常考角度
- 上午题常考语法语义、参数传递、作用域、递归、控制结构和面向对象基本概念。
- 这部分是后续系统分析、设计与建模题的基础，尤其影响下午题表述理解。
- 旧教材里的语言细节不必死抠，重点是抽象机制与程序组织思想。

## 互联网重点标注：中
- 互联网备考普遍把本章定位为“分析设计语言基础”，不会像程序员考试那样深挖语法细节。
- 结合新版教材趋势，应重点把握抽象建模、结构化与面向对象思想，而不是具体语言语法。

## 复习建议
- 把过程式与面向对象的核心差异背熟，便于后续 UML、设计题快速上手。
- 用少量伪代码题验证递归、参数传递、作用域等概念是否真正理解。
