# 第12章 信息系统分析

## 教材定位
- PDF 目录起始页：238
- 目录关键词：系统分析任务、分析原则、结构化分析、说明书、UML基础

## 你要掌握的核心知识
- 掌握系统分析的任务、原则和工作产出，重点是需求获取、建模和确认。
- 理解结构化分析方法、数据流分析、逻辑模型和需求规格说明书的编制。
- 掌握系统说明书、模型表达和 UML 基本概念在分析阶段的用途。
- 理解需求完整性、一致性、可验证性和可追踪性的要求。

## 常考角度
- 下午题高频考点包括需求调研、业务建模、需求规格、分析文档和 UML 基础。
- 题目常让你判断需求描述是否完整，或要求补充分析阶段应做的工作。
- 系统分析与数据库设计、系统设计、实施验收紧密相连。

## 互联网重点标注：高
