# 第13章 信息系统设计

## 教材定位
- PDF 目录起始页：274
- 目录关键词：系统设计概述、结构化设计、系统结构图、数据库设计、输入输出设计、体系结构

## 你要掌握的核心知识
- 掌握系统设计目标、原则以及从逻辑模型走向物理实现的设计过程。
- 理解结构化设计和面向对象设计的基本方法、模块划分和耦合内聚思想。
- 掌握数据库设计、输入输出设计、处理过程设计和系统文档编制。
- 理解系统总体技术方案和体系结构设计的思路。

## 常考角度
- 下午题高频考点包括模块划分、结构图、数据库设计、输入输出设计和体系结构说明。
- 要会从需求出发说明为什么这样设计，而不是只会背方法名。
- 设计题喜欢考质量属性权衡，如性能、可靠性、安全性、扩展性。

## 互联网重点标注：高
- 网上备考普遍认为设计章是下午题拉开差距的关键章节。
- 结合新版教材新增的架构内容，建议把本章升级为‘需求 -> 架构 -> 模块 -> 数据 -> 接口’一条线来复习。
