# 第16章 系统管理规划

## 教材定位
- PDF 目录起始页：378
- 目录关键词：系统管理定义、需求分析、IT人员需求、制度与规划、用户沟通

## 你要掌握的核心知识
- 掌握系统管理的定义、目标和管理需求来源。
- 理解从用户视角和 IT 人员视角进行管理需求分析的方法。
- 掌握管理规划、制度建设、角色职责和实施路径设计的思路。
- 理解规划阶段如何为后续资源、故障、安全、性能等专题管理奠定框架。

## 常考角度
- 管理需求分析、管理制度规划、职责边界和实施路线是常见考点。
- 下午题喜欢考‘怎么从业务目标推导管理机制’。
- 本章是第三篇系统管理各专题的总入口。

## 互联网重点标注：高
- 近年备考资料普遍把系统管理规划列为高优先级，因为它直接决定运维治理题的答题框架。
- 结合新版考试更重视 IT 服务与治理的趋势，本章建议用‘目标、流程、角色、制度、指标’五个维度复习。

## 复习建议
- 做题时先分清是规划层问题还是执行层问题，答题更稳。
- 把规划对象和规划输出物整理清楚，如制度、流程、职责、指标、计划。
