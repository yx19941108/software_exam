# 第25章 系统用户支持

## 教材定位
- PDF 目录起始页：537
- 目录关键词：用户视角、文档、咨询、帮助服务台、人员培训与服务

## 你要掌握的核心知识
- 掌握系统用户支持的目标、范围和价值，理解其对服务质量和用户满意度的影响。
- 理解用户文档、培训、咨询、帮助台、知识库和服务流程的建设要点。
- 掌握用户问题受理、分级、跟踪、反馈和满意度管理。
- 理解用户支持与故障管理、维护管理、评价改进之间的关系。

## 常考角度
- 帮助台、培训、文档、服务流程和满意度管理是常考点。
- 下午题经常要求从用户视角提出支持与保障措施。
- 答题时要体现可达性、响应性、闭环和持续改进。

## 互联网重点标注：高
- 当前网上备考普遍把用户支持视为运维管理场景题的重要落点。
- 结合新版教材更强调信息技术服务，本章建议重点掌握服务台、知识库、SLA、满意度和反馈闭环。

## 复习建议
- 复习时把‘用户问题从提出到关闭’的流程画出来，帮助形成答题框架。
